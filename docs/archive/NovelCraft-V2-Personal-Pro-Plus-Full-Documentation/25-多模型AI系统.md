# 多模型 AI 系统

## 概述

支持 4 家 AI Provider 的统一网关，实现模型路由、降级、对比和成本控制。

## 支持 Provider

| Provider | 模型 | 适用场景 |
|---|---|---|
| **DeepSeek** | deepseek-chat / deepseek-coder | 草稿/摘要/字段生成（高性价比） |
| **Claude** | claude-sonnet-4 / claude-opus-4 | 关键章节/重写/风格化（文学质量最高） |
| **OpenAI** | gpt-4o / gpt-4o-mini | 通用/复杂推理 |
| **Gemini** | gemini-2.0-pro | 长上下文备胎（100 万 token 窗口） |

## AI Gateway 统一入口

```python
gateway.complete(
    task_type: str,       # gen_chapter / review / polish / translate...
    prompt_name: str,     # Prompt 库中的名称
    variables: dict,      # 模板变量
    output_schema: Type,  # Pydantic 输出校验
    budget_scope: str     # 预算范围（task/project/daily）
) -> AIResult
```

执行流程：
1. 预算检查（Redis 计数，超限 → PENDING_BUDGET）
2. 路由（model_routes 表查 task_type → provider/model）
3. Prompt 渲染（Jinja2，带 version + model 分支）
4. Provider 调用（令牌桶限流 + 指数退避重试）
5. 降级链（主失败 → 备 1 → 备 2 → PENDING_PROVIDER）
6. Schema 校验（失败带错误重试 ≤ 2 次）
7. 全量落库 ai_calls（九要素完整）

## 模型路由

```sql
model_routes(
  task_type,           -- gen_chapter / review / polish...
  primary_provider,    -- DeepSeek
  primary_model,       -- deepseek-chat
  fallbacks JSONB,     -- [{"provider":"Claude","model":"claude-sonnet-4"}, ...]
  params JSONB         -- temperature / max_tokens / top_p
)
```

路由配置热更新（Redis 缓存），无需重启服务。

## 交叉审核策略

- 生成：DeepSeek（快速/便宜）
- 审核：Claude（高文学判断力，与生成异构）
- 关键场景（黄金三章/出海翻译）：用最强模型

## A/B 测试

- model_routes 支持百分比分流
- 效果通过 ai_calls 聚合（质量分 + 成本 + 延迟）
- A/B 报表自动对比

## Token 预算

三级熔断（Redis 计数 + 落库对账）：
- 单任务：如 bootstrap 上限 ¥2
- 单项目：如项目日上限 ¥50
- 单日：全局日上限 ¥200

超限不失败，置 PENDING_BUDGET，用户可手动调整。

## 成本面板

从 ai_calls 聚合，无独立记账系统：
- 单章成本
- 单篇内容成本
- 按模型/Agent/任务类型聚合
- 趋势图（日/周/月）
