# 自动生产流水线

## 概述

从选题到发布的全自动内容生产线，由工作流引擎驱动。

## 核心流水线

### 长篇连载流水线
```
定时触发 → 细纲生成 → 细纲审核(human) → 正文生成 → 7维审核
  → 质量分<80? → 自动返工(≤2轮) → 终审(human) → 发布(可选)
```
- 支持"生成 N 章后暂停等待审核"
- 支持"卷级复盘门禁"（每卷完成必须人工确认）

### 短篇批量流水线
```
选题输入 → 批量生成(同主题×5篇) → 批量审核 → 选择最佳 → 发布
```

### 一稿多平台 Fan-out
```
源内容 → 生成公众号版 → 生成头条版 → 生成小红书版 → 生成知乎版
  → 生成Medium版 → 全部等待审核(human) → 逐平台发布
```

### 每日热点晨报
```
beat 09:00 → 抓热点 → AI分析 → 选题推荐 → 生成3平台草稿 → 等待审核
```

### 自动连载
```
beat 定时 → 检查是否有待续写小说 → 生成下一章 → 审核 → 可选自动发布
```

## 执行保障

- **幂等**：同一次 run 的同一节点不会重复执行
- **断点续跑**：系统重启后从上次成功节点继续
- **Human 挂起**：等待审核不占用 worker，恢复后继续
- **失败重试**：节点级自动重试（≤ 2 次），可单点重跑
- **预算熔断**：超预算自动暂停，置 PENDING_BUDGET
- **告警**：失败/超预算/Provider 全降级 → Telegram 通知

## 监控面板

- 当前运行的流水线状态（实时 SSE）
- 历史执行记录（成功/失败/暂停/取消）
- 执行日志（每节点输入/输出/耗时）
- 成本统计（每条流水线的 AI 调用费用）
