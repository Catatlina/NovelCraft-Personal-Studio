# 测试验收标准

## 测试策略

分层测试金字塔：
```
        E2E（关键路径）
      ┌──────────────┐
      │ 集成测试      │
    ┌─┴──────────────┴─┐
    │    单元测试       │
  └─┴──────────────────┴─┘
```

- **单元测试**：pytest + pytest-asyncio，覆盖率 ≥ 80%
- **集成测试**：API 端点 + DB + Redis
- **E2E**：关键用户旅程（bootstrap/连载/发布）
- **性能测试**：30 万字连载压测

## Golden Case 回归（CI）

每个 Agent 至少 1 条 golden case：
```python
def test_writer_golden():
    result = agent_registry["Writer"].execute(GOLDEN_INPUT)
    assert result.word_count >= 2000
    assert result.has_dialogue
    assert schema_valid(result)
```

## M1 MVP 验收门禁

1. 新用户 ≤ 15 分钟灵感 → 过审第一章
2. 杀 worker/断网后 run 断点续跑
3. human 节点挂起 24h 后确认仍可继续
4. 单次 bootstrap 成本 ≤ ¥2，超限熔断 PENDING_BUDGET
5. 任意保存/AI 重写可版本恢复
6. ai_calls 九要素完整记录
7. V1 数据迁移校验通过且可回滚
8. 单测新增 ≥ 30，总 ≥ 135 全绿
9. lint/type/build 过
10. 备份恢复演练通过
11. 失败告警实收测试通知

## M2 验收

- 30 万字连载压测通过
- 人工抽检一致性错误率 < 5%
- 7 维均分 ≥ 80
- 伏笔到期回收提醒与校验生效
- 自定义工作流可成功编排执行
- 4 家模型路由与降级演练通过
