# 部署运维文档

## 部署架构

单 VPS（4C8G 起步）+ Docker Compose 全栈。

```
Nginx (HTTPS) → FastAPI (uvicorn×2) → PostgreSQL + pgvector
                                    → Redis
              → Celery Workers ×5   → PostgreSQL
              → Celery Beat         → PostgreSQL
```

## Docker Compose 服务

| 服务 | 镜像 | 说明 |
|---|---|---|
| nginx | nginx:1.27-alpine | TLS 终止 · 反代 · SSE · 静态 |
| api | novelcraft/api | uvicorn×2 |
| worker-plan | novelcraft/worker | celery -Q plan |
| worker-generate | novelcraft/worker | celery -Q generate |
| worker-review | novelcraft/worker | celery -Q review |
| worker-publish | novelcraft/worker | celery -Q publish |
| worker-system | novelcraft/worker | celery -Q system |
| beat | novelcraft/worker | celery beat |
| postgres | pgvector/pgvector:pg16 | 唯一事实源 + 向量 |
| redis | redis:7-alpine | broker · 限流 · 预算 · 缓存 · SSE |

## 首次部署

```bash
# 1. 配置环境变量
cp .env.example .env
vim .env  # 填入密钥/API Key/域名

# 2. 启动服务
docker compose up -d

# 3. 运行迁移
docker compose exec api alembic upgrade head

# 4. 加载 seed 数据
docker compose exec api python -m app.seed

# 5. 配置 Nginx + HTTPS (certbot)
```

## 备份策略

- **频率**：每日 02:00（beat 定时任务）
- **方式**：pg_dump | age 加密 → rclone 异地（对象存储）
- **保留**：本地 7 天 + 异地 30 天
- **RPO**：≤ 24h
- **RTO**：≤ 1h
- **验证**：每月恢复演练

## 健康检查

`GET /healthz` 聚合检查：
- PostgreSQL 连接
- Redis 连接
- 队列深度
- 磁盘使用率
- 各 Provider API 连通性

异常 → Telegram Bot 告警。

## 告警规则

| 事件 | 通知方式 |
|---|---|
| 任务失败 | Telegram |
| 预算触顶 | Telegram |
| Provider 全降级 | Telegram |
| 磁盘 > 80% | Telegram |
| 备份失败 | Telegram |
| 健康检查异常 | Telegram |

## 日志

structlog JSON 格式 → 本地滚动 + grep。
1~5 人规模不上 ELK。

## 升级流程

```bash
make deploy
# = docker compose pull
# + 自动备份
# + docker compose up -d
# + alembic upgrade head
# + healthcheck
# + 失败自动回滚上一 tag
```
