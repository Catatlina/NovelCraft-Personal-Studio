## 三条铁律

1. **LLM 调用只准走 Gateway**：CI 静态检查 `grep -r 'openai\|anthropic\|google.generativeai' --include='*.py' app/ | grep -v 'app/ai/'` 必须为空
2. **写操作只准走 VersionedRepository**：任何修改 content/knowledge/prompt/workflow 必须先快照
3. **内容读写只准走 content_service**：meta JSONB 写入前必须过 Pydantic Schema 校验

## 编码规范

- Python 3.11+，类型注解全覆盖（mypy strict）
- 异步优先（async/await），阻塞操作放 Celery
- Pydantic v2 做所有输入输出校验
- structlog 结构化日志（JSON 格式）
- Alembic 线性迁移（一 PR 一迁移）
- pytest + pytest-asyncio（覆盖率 ≥ 80%）

## 错误处理

- 统一异常处理中间件
- 错误码规范：`{模块}_{错误类型}`（如 CONTENT_NOT_FOUND, BUDGET_EXCEEDED）
- AI 调用失败分级：重试 → 降级 → PENDING_PROVIDER → 告警

## Git 规范

- 分支：feature/ fix/ chore/ 前缀
- Commit：conventional commits（feat/fix/chore/docs/test）
- PR 必须通过 CI（lint + type + test）方可合入
