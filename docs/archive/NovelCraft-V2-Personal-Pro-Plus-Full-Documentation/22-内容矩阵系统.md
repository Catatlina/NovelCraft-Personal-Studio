# 内容矩阵系统

## 概述

管理创作者的跨平台内容矩阵，一篇文章的多平台版本统一管理、追踪和优化。

## 核心概念

**内容矩阵**：同一主题/素材在不同平台上的内容集合。

```
源内容（如一篇深度长文）
  ├─ 公众号版
  ├─ 头条版
  ├─ 小红书版（笔记 × 3）
  ├─ 知乎版（问答 + 文章）
  ├─ 百家号版
  ├─ Medium版
  ├─ Substack版
  └─ X Thread版
```

## 血缘追踪

`derivations` 表记录内容复用关系：
- `source_content_id` → `derived_content_id`
- `workflow_run_id` 记录哪次工作流产生的派生
- 可视化血缘图（树状/网状的派生关系）

## 矩阵管理页

- **列表视图**：按时间/平台/状态排序
- **看板视图**：按平台分列，拖拽卡片
- **筛选**：按平台/状态/日期/标签
- **批量操作**：批量发布/批量删除/批量重写

## 发布状态追踪

每个矩阵内容跟踪：
- 生成状态：草稿/已生成/已审核/已发布
- 发布平台/发布时间/发布链接
- 数据回流：阅读/点赞/收藏/评论/分享
- 优化建议：哪些平台表现好/哪些需要调整

## ROI 分析

- 单篇内容的跨平台总收益
- 平台横向对比（哪个平台 ROI 最高）
- 内容类型对比（哪种内容类型最受欢迎）
