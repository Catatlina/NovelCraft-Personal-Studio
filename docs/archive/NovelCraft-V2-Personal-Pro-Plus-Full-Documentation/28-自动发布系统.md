# 自动发布系统

## 概述

多平台内容发布网关，支持手动/半自动/全自动三种模式。

## 发布三模式

| 模式 | 适用平台 | 操作方式 | 风险 |
|---|---|---|---|
| **手动** | 全部平台 | 排版 + 一键复制，人工去平台粘贴发布 | 零风险 |
| **半自动（默认）** | 公众号/头条/小红书/知乎/百家等 | Playwright 自动填充内容，**人点最终"发布"按钮** | 低风险 |
| **全自动** | 仅官方 API：Medium/Substack/X/WordPress | 系统直接调用官方 API | 官方授权合规 |

## 国内平台全自动的强约束

1. **默认关闭**：需在设置页显式开启
2. **二次确认**：开启时弹窗"我已知悉平台条款与封号风险"，操作写 audit_logs
3. **首篇强制 human 节点**：每平台第一篇仍需人工确认
4. **连续失败降级**：触发验证码/风控 → 自动降级为半自动 + 告警

## PublisherAdapter 接口

```python
class PublisherAdapter(ABC):
    async def format(content: Content) -> FormattedContent
    async def publish(content: FormattedContent, account: PlatformAccount) -> PublishResult
    async def fetch_metrics(content_id: str) -> MetricsData
```

每平台一个 adapter，统一接口。首批 adapter：
- 全自动：Medium / Substack / X API / WordPress
- 半自动：公众号 / 头条 / 小红书 / 知乎 / 百家号 / 大鱼号 / 网易号

## 发布前安全检查

- 敏感词过滤（sensitive_words 表 + 平台规则库）
- 仿写相似度标记拦截（≥ 0.75 禁止发布）
- 平台内容规范校验

## 发布记录

```sql
publish_records(
  content_id, platform, account_id,
  status,              -- pending/published/failed
  published_url,
  scheduled_at,        -- 定时发布时间
  published_at,
  error, risk_notice
)
```

## 数据回流

三级回流策略：
1. **API 自动**：Medium/Substack/X → 定时拉取阅读/点赞/评论
2. **半自动**：Playwright 抓取 + OCR 识别
3. **手动兜底**：手动录入阅读/点赞/收藏/评论/分享/收益

metrics 表按 content × platform × date 聚合。
