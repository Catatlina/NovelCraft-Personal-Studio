# 数据库 ER 关系说明

## 核心实体关系

```
users ──< project_members >── projects
  │                                 │
  │                          ┌──────┼──────┬──────────┐
  │                          ▼      ▼      ▼          ▼
  │                      contents knowledge workflows budgets
  │                          │      │      │
  │              ┌───────────┤      │      ├──── schedules
  │              ▼           ▼      ▼      ▼
  │        derivations   versions  knowledge_vectors
  │              │                    │
  │              ▼                    ▼
  │         contents           workflow_runs
  │           (derived)              │
  │                                  ▼
  │                              run_nodes
  │                                  │
  │                                  ▼
  │                              ai_calls
  │
  ├── platform_accounts ──< publish_records >── metrics
  ├── prompts
  ├── model_routes
  └── audit_logs
```

## 关键关系说明

### Content 树
`contents.parent_id` 自引用，形成内容树：
- novel → volume → chapter（小说结构）
- source_content → derived_content（派生血缘，通过 derivations 表）

### 版本系统
`versions` 为多态表：
- `entity_type` = 'content' | 'knowledge_item' | 'prompt' | 'workflow'
- `entity_id` 指向对应实体
- `parent_version_id` 形成版本树（分支）

### 工作流链
workflows → workflow_runs → run_nodes → ai_calls
一 run 多 node，一 node 多 ai_call

### 知识检索
knowledge_items ──< knowledge_vectors（1 item : N chunks）
pgvector HNSW 索引支持 `kb.search()`

### 叙事一致性
contents(chapter) ──< entity_states
contents(chapter) ──< foreshadowings
contents(chapter) ──< timeline_events
contents(novel)   ──< arcs

### 发布回流
platform_accounts ──< publish_records ──< metrics
每篇 content 可按多平台发布，每平台多次发布记录，每次发布可有多日数据回流

### 权限
users ──< project_members(role: owner/editor/viewer) >── projects
所有项目内实体通过 project_id 归属，Repository 基类强制 `WHERE project_id = ?`
