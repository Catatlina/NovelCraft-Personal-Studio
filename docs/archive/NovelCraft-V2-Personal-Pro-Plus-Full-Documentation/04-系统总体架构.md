# 系统总体架构

## 架构图

```
React + Vite + TS（PWA · 设计系统 · Tiptap 三栏工作台）
        │  REST + SSE（进度/通知）
FastAPI（api → service → repository · JWT + Owner/Editor/Viewer）
        │
├── PostgreSQL + pgvector   唯一事实源（内容/知识/版本/追踪/向量）
├── Redis                   Celery broker · 限流 · 预算计数 · 缓存 · SSE 发布
├── Celery + beat           五队列（plan/generate/review/publish/system）+ 定时自动化
├── AI Gateway（进程内模块） 4 家 Provider 路由/降级/预算/Schema 校验/全量追踪
└── Nginx + HTTPS · Docker Compose · 单 VPS
```

## 架构原则

1. **单 VPS 垂直扩容优先** — 不引入微服务/K8s/分布式
2. **PostgreSQL 唯一事实源** — Redis 仅作缓存/队列，数据可丢弃重建
3. **进程内模块** — 工作流引擎、发布网关、Knowledge Hub 均为 FastAPI 进程内模块 + Celery 任务
4. **离线优先** — PWA + IndexedDB + Service Worker
5. **Agent = 工作流节点** — 不做自治对话式多 Agent

## 8 大平台能力（C1~C8）

| 能力 | 名称 | 职责 |
|---|---|---|
| C1 | 统一内容模型 | 单 `contents` 表承载所有体裁 |
| C2 | 工作流引擎 | 四节点类型（agent/human/tool/branch）编排一切自动化 |
| C3 | AI Gateway + Agent | 4 Provider 路由/降级/预算/全量追踪 |
| C4 | Knowledge Hub | 素材中心 + pgvector RAG + 四库 + 风格学习 |
| C5 | 通用版本系统 | 多态版本树，覆盖内容/人物/世界观/Prompt/知识 |
| C6 | 叙事一致性引擎 | 7 层上下文 + 伏笔/时间线/弧线/防崩 |
| C7 | 发布网关 | 三模式发布 + 数据回流 + ROI |
| C8 | 追踪与治理 | ai_calls 全记录 + 审计日志 |
