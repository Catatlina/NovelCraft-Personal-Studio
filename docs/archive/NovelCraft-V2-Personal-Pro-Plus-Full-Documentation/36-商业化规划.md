# 商业化规划

> NovelCraft Personal Studio 当前定位为个人/小团队自托管工具，商业化是**远期可选方向**，不在当前开发范围内。

## 当前定位

- **自托管**：用户在自己的 VPS 上部署，数据完全自主
- **开源**：代码开源（GitHub），社区贡献
- **免费**：用户只需支付自己的 AI API 费用

## 商业化触发条件

以下条件**同时满足**时，才考虑商业化：

1. ROI 面板显示持续正收益（用户确实通过工具赚到钱）
2. 出现真实外部用户需求（非开发者本人）
3. 功能稳定、运维成熟（M5 完成后）
4. 有明确的 SaaS 化路径而不需要推倒重来

## 可能的商业化方向（远期）

### SaaS 托管版
- 用户无需自己部署 VPS
- 按月/年订阅
- 不同套餐（按内容量/功能/协作人数）

### 企业版
- 5~20 人团队
- 高级协作功能
- 专属模型调优
- SLA 保障

### API 服务
- 叙事一致性引擎 API
- 工作流引擎 API
- 多平台发布 API

## 明确不做（即使商业化）

- ❌ 不改成微服务架构
- ❌ 不引入 K8s
- ❌ 不做复杂的多租户隔离
- ❌ 不做支付计费系统（用现成的 Stripe/LemonSqueezy）
- ❌ 不做企业组织架构/SSO
- ❌ 不在内容中插入广告

## 架构兼容性

当前 V2.1 架构已为可能的 SaaS 化预留空间：
- `projects` 表天然隔离用户数据
- `project_members` 已支持多用户
- `budgets` 已支持项目级预算
- UUIDv7 主键天然支持分布式
