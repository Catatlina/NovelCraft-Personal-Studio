# 技术选型说明

## 选型总览

| 层 | 选型 | 理由 |
|---|---|---|
| 前端框架 | React 18 + TypeScript | 生态成熟，组件化，Tiptap/Radix 兼容好 |
| 构建工具 | Vite | 快速 HMR，ESM 原生 |
| UI 组件 | Radix UI + shadcn/ui + Tailwind CSS | 无头组件，定制灵活，暗色原生 |
| 编辑器 | Tiptap | 基于 ProseMirror，扩展性强，JSON 输出 |
| 状态管理 | Zustand + TanStack Query | 轻量，服务端态/客户端态分离 |
| PWA | vite-plugin-pwa | Service Worker + IndexedDB |
| 后端框架 | FastAPI (Python 3.11+) | 异步原生，Pydantic 校验，OpenAPI 自文档 |
| 数据库 | PostgreSQL 16 + pgvector | ACID + 向量检索，单库解决 |
| 缓存/队列 | Redis 7 | Celery broker + 限流 + 预算计数 + SSE 发布 |
| 任务队列 | Celery + beat | Python 生态标准，五队列隔离 |
| AI Gateway | 进程内模块 | 不引入额外服务，维护成本恒定 |
| 反向代理 | Nginx | TLS 终止 + SSE + 静态文件 |
| 容器化 | Docker Compose | 单 VPS 标准方案 |
| 备份 | pg_dump + age 加密 + rclone | 简单可靠 |
| 监控告警 | structlog + Telegram Bot | 轻量，1~5 人够用 |

## 明确不引入

| 组件 | 理由 |
|---|---|
| Kubernetes | 1~5 人不需要编排层 |
| Elasticsearch | pgvector + tsvector 够用 |
| Kafka/RabbitMQ | Redis 做 broker 节省运维 |
| Prometheus/Grafana | Telegram 告警够用，不上全套监控 |
| ELK | structlog + grep 够用 |
| Nginx Plus/商业版 | 开源版功能足够 |
| gRPC | REST + SSE 满足需求 |
| GraphQL | REST 更简单直接 |

## 关键设计决策

1. **UUIDv7 为主键**：支持离线创建（客户端先行生成 ID）
2. **JSONB 用于半结构化**：content meta / workflow definition / ai_calls input-output
3. **Tiptap JSON 为正文格式**：统一编辑格式，按平台序列化输出
4. **Alembic 线性迁移**：一 PR 一迁移，可升降
5. **五队列隔离**：plan/generate/review/publish/system，避免互相影响
