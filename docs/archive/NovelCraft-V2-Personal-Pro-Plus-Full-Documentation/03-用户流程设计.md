# 用户流程设计

## 1. 核心用户旅程

### 1.1 新用户引导（First Run）
注册/登录 → 创建项目 → 输入灵感（一句话/一段 + 题材/风格/目标字数）
→ bootstrap 工作流：书名(3候选) → 简介卖点 → 世界观 → 人物(3~6) → 总纲 → 第一章 → 7维审核
→ 审阅页（选标题/确认或退回重跑）→ 编辑器修改 → 保存（自动版本快照）

目标：≤15 分钟从灵感到过审第一章

### 1.2 连载作者日常
打开项目 → 查看当前章节审核意见 → 一键续写/手动编辑 → 提交审核 → 发布
→ 查看数据反馈 → 调整下章方向

### 1.3 自媒体矩阵运营
抓热点 → 分析选题 → 一稿多平台生成 → 逐平台审核 → 定时发布 → 查看数据回流 → ROI 分析

## 2. 工作流引擎交互

### 2.1 预设工作流
8 条预设工作流覆盖核心场景：
1. 长篇章节流水线
2. 黄金三章
3. 短篇爆款
4. 一稿五平台
5. 每日热点晨报
6. 自动连载
7. 出海翻译发布
8. 拆书入库

### 2.2 人工确认节点
- 工作流到达 human 节点时挂起
- 前端弹出确认卡片（如"请选择书名"三选一）
- SSE 实时推送进度
- 24h 后挂起仍可继续确认

### 2.3 可视化编排
- DAG 编辑器（拖拽节点/连线）
- 节点开关/排序/模型与 Prompt 选择
- 暂停/恢复/取消/单点重试

## 3. 编辑器交互

### 3.1 三栏布局
- 左侧：项目树/目录/人物/设定
- 中间：Tiptap 编辑器（正文）
- 右侧：AI 助手面板/审核意见/版本历史

### 3.2 AI 交互点
- 选中浮条：续写/改写/润色/扩缩写/去AI味
- 右键菜单：AI 操作
- ⌘K：全局命令（创建内容/搜索/切换项目/AI 操作）
- 空白行：AI 补全提示
- 审核页：7 维雷达图 + 问题清单 + 一键重写

### 3.3 版本管理
- 自动保存版本（每分钟快照）
- AI 重写版本（分支树）
- 手动保存版本
- 任意版本恢复/对比

## 4. 离线交互

- L1 只读：断网可浏览项目树/章节/人物/知识/版本
- L2 编辑：离线编辑本地暂存，恢复网络自动提交
- L3 出站：AI 任务排队，恢复网络自动执行
