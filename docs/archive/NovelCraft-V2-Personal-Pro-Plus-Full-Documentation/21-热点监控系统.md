# 热点监控系统

## 概述

实时监控全网热点，AI 分析价值并推荐选题，驱动内容生产决策。

## 数据源（可插拔 Adapter 模式）

### 新闻热榜
- 微博热搜
- 百度热榜
- 知乎热榜
- 今日头条热榜

### 行业热点
- AI / 科技 / 财经 / 商业专题
- RSS 订阅源
- 行业媒体 API

### 内容平台榜单
- 起点中文网（畅销/月票/推荐）
- 番茄小说
- 小红书趋势
- 抖音热点

## Adapter 接口

```python
class TrendAdapter(ABC):
    async def fetch_trends() -> list[TrendItem]
    async def get_detail(url: str) -> TrendDetail

# 每个数据源一个 adapter，失败自动降级
adapters = [WeiboAdapter(), BaiduAdapter(), ZhihuAdapter(), ...]
```

> 榜单接口随时变，adapter 做成可插拔，失效时降级为手动录入。

## AI 分析能力

| 分析 Agent | 输入 | 输出 |
|---|---|---|
| 价值评估 | 热点条目 | 热度分/持续度/相关性/商业价值 |
| 平台适配 | 热点 + 目标平台 | 适配建议/角度推荐 |
| 选题推荐 | 热点分析结果 | 选题列表 + 角度 + 标题建议 |
| 爆款分析 | 爆款内容 | 开篇结构/爽点/节奏/标签/传播潜力 |
| 趋势分析 | 时间序列数据 | 上升/下降/周期趋势 |

## 选题库

- AI 推荐的选题自动入库
- 手动添加选题
- 选题状态：待评估 → 已采纳 → 生产中 → 已完成
- 选题关联内容（追踪从选题到发布的全链路）

## 每日热点晨报（定时工作流）

```
每天 09:00 自动运行:
抓热点 → AI 分析 → 选题推荐 → 生成多平台草稿 → 等待审核
```

- 连续 7 天无人值守成功（human 节点等待审核）
- 失败自动告警
