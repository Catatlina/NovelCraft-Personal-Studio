# 安全设计

## 概述

覆盖认证授权、数据加密、AI 安全和上线检查清单。

## 认证与授权

### JWT
- 短期 access token（15 min）+ 长期 refresh token（7 days）
- 密钥通过环境变量注入
- Token 黑名单（Redis，登出即失效）

### RBAC（极简三角色）
- **Owner**：全权（删项目/管成员/改预算/改路由）
- **Editor**：读写内容/跑工作流；不可删项目/管成员/改预算
- **Viewer**：只读

API 层装饰器校验 + ProjectScopedRepository 基类强制 `WHERE project_id = ?`

## 数据加密

- **API Key**：Fernet 对称加密存储（platform_accounts 表）
- **平台账号密码**：Fernet 加密
- **JWT Secret**：环境变量，不落库
- **备份**：age 加密后异地传输

## Web 安全

| 威胁 | 防护 |
|---|---|
| CSRF | SameSite=Strict Cookie + CSRF Token |
| XSS | Tiptap schema 白名单 + 输出 HTML 转义 |
| SQL 注入 | SQLAlchemy ORM 参数化 + CI 裸 SQL 检查 |
| 敏感信息泄露 | 错误响应不暴露堆栈/内部路径 |

## AI 安全

- **注入防护**：用户输入与 Prompt 模板分离（Jinja2 变量注入，不是字符串拼接）
- **内容安全过滤**：发布前过敏感词表 + 平台规则库
- **仿写防侵权**：相似度 ≥ 0.75 拦截，0.6~0.75 人工确认
- **预算熔断**：防止恶意调用耗尽余额
- **速率限制**：API 端点级令牌桶限流（Redis）

## 审计日志

```sql
audit_logs(
  user_id, action,        -- 谁做了什么
  entity_type, entity_id, -- 对什么
  details JSONB,          -- 变更细节
  ip_address, user_agent,
  created_at
)
```

中间件级记录，关键操作（删除/权限变更/预算修改/全自动发布开启）强制写 audit_logs。

## 上线前安全检查清单

- [ ] JWT Secret 强度 ≥ 256 bit
- [ ] 所有 API Key 已加密存储
- [ ] HTTPS 已配置（TLS ≥ 1.2）
- [ ] 数据库密码不为默认值
- [ ] Redis 密码已设置
- [ ] CORS 白名单已限定
- [ ] 错误响应不含堆栈信息
- [ ] 备份已测试恢复
- [ ] 告警通道已测试
