# Context Hub 记忆系统

## 概述

Context Hub 是叙事一致性引擎（C6）的核心组件，负责为每次 AI 生成装配上下文。目标：让 AI 在写第 100 章时仍然"记得"第 1 章发生了什么。

## 7 层上下文装配

按优先级从高到低，Token 预算制，超限从低优先级丢弃：

| 层级 | 内容 | Token 预算 | 优先级 |
|---|---|---|---|
| ① 全书状态摘要 | 当前进度/主要人物状态/未回收伏笔 | ≤ 1000 | 最高 |
| ② 本卷摘要 | 当前卷的核心事件/人物变化 | ≤ 800 | 高 |
| ③ 近 3 章摘要 | 最近章节的详细摘要 | ≤ 600 | 高 |
| ④ 实体状态表切片 | 本章出场人物的位置/关系/持有物/已知信息 | ≤ 500 | 中 |
| ⑤ 到期伏笔/时间线提醒 | 计划在本章回收的伏笔/关键时间节点 | ≤ 300 | 中 |
| ⑥ RAG 召回素材 | 从 Knowledge Hub 检索的相关设定/金句/风格卡 | ≤ 1500 | 低 |
| ⑦ 本章细纲 + 写作规则 | 章节写作指导 + 风格约束 | ≤ 700 | 最低 |

总计 ≤ 5400 tokens，硬截断 + 低优先级先丢。

## 分层摘要系统

- **章级摘要**：每章完成自动生成（人物动作/情节推进/新伏笔/设定变更）
- **卷级摘要**：每卷完成自动聚合（核心冲突/人物弧线进展/回收伏笔）
- **全书状态摘要**：全局视图，每次更新后刷新

## 实体状态表

```sql
entity_states(
  chapter_id, entity_type,  -- character/location/item/faction
  entity_name,
  location,                 -- 当前位置
  relationships JSONB,      -- {"张三": "盟友", "李四": "敌对"}
  possessions JSONB,        -- ["青冥剑", "藏宝图"]
  known_info JSONB,         -- {"秘密X": true, "阴谋Y": false}
  updated_at
)
```

逐章更新，硬约束注入生成上下文——防止"上章在京城、这章突然在边疆"的崩坏。

## 装配结果追踪

每次装配的完整快照随 `ai_calls.input` 落库，防崩问题可回放定位"当时模型看到了什么"。
丢弃日志记录哪些层级被截断，辅助调整预算分配。
