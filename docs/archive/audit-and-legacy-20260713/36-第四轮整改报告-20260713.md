## 二、审计过程中发现并一并修复的门面/缺陷

1. **fusion.py 路由未挂载**——`53a3857` 提交的 `/fusion/status` 从未 include，恒 404。已挂载 `/api/v1/fusion/status`，实测 200（`tests/test_fusion_wired.py`）。
2. **前端 Progress 页仍找旧节点键**——书名门找 `n2`、策划产物找 `n3-n6`，V2 下人工确认门**永远不显示**（旅程 A 即使后端跑通也会卡在 UI）。已改为 `human_confirm_title` + V2 节点键（兼容旧 run）。
3. **书名候选无来源**——V2 规划 7 节点没有任何一个产出 `title_candidates`，人工门无可选项。已将书名候选并入 `plan_idea` 输出契约（5 个候选），经 `context.update(output)` 自动进 run context，前端直接可渲染。
4. **DB 包装器缺 `rowcount`**——`execute_bootstrap` 的抢占守卫 `hasattr(claim,"rowcount")` 恒 False，形同虚设；已补属性（连带使 QA-004 删除接口能统计级联数）。
5. **ProviderError 落库丢失真实错误**——节点错误固定写"provider error"，线上排障（含上一轮 QA-001）盲。已透传真实 provider 消息。
6. **_mock_output 不覆盖 V2 节点**——mock 通道跑 V2 流必然 schema 失败。已补 18 个 schema 合规 mock（使 mock T2 走与真实 Provider 相同的校验管道）。
