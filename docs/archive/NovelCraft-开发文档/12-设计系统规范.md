# NovelCraft Personal Studio · 设计系统规范

> 版本：V1.0 ｜ 适用范围：前端全量 UI（React + Vite + TS，PWA）
> 关联文档：02-架构评审报告 §9（前端/离线、设计系统先行、AI 无处不在）；05-技术实施方案 §2（前端结构与设计系统）
> 一句话原则：**UI 第一。** 对标 Linear / Notion / Cursor / Raycast，暗色模式原生。

---

## 0. 总纲

设计系统是 MVP 前 1.5~2 周必须先行的底座（02 §9 定案）。本规范定义：

- **设计原则**与对标基准
- **Design Tokens**（色彩/字号/间距/圆角/阴影/层级/动效，明暗双套，含具体值）
- **暗色模式实现**
- **组件库清单**（基于 Radix / shadcn 定制）
- **布局 / 编辑器专属组件 / AI 交互标准 / UI 验收 / a11y 基线**

所有视觉决策必须来自 tokens，**禁止在业务代码里写裸色值、裸间距、裸字号**（见 §8 stylelint 守卫）。

## 1. 设计原则与对标基准

### 1.1 UI 第一

创作者把 NovelCraft 当"写作环境"而非"后台系统"。每一处交互成本都要低于"直接开 Word"。具体：

- 默认进入即工作状态（无冗余引导）；常用操作 ≤2 次点击或 1 个快捷键。
- 信息密度高但不拥挤（留白用 tokens 控，不靠感觉）。
- 任何 AI 能力都不打断心流：浮条、⌘K、右键菜单，而非弹窗流。

### 1.2 对标基准（三向取法）

| 维度 | 对标产品 | 学什么 |
|---|---|---|
| 信息密度 | **Linear** | 列表/表格紧凑、行高克制、状态用色点而非文字、键盘优先 |
| 编辑沉浸 | **Notion / Cursor** | 块编辑流畅、选中即出 AI 浮条、无模式切换的 AI 内联 |
| 命令交互 | **Raycast** | 全局 ⌘K 命令面板、模糊搜索、动作即命令 |

### 1.3 设计价值观

1. **Calm by default**：默认安静（低饱和中性底），高亮只给"此刻重要的事"。
2. **Dense but breathe**：高密度 + 规律留白。
3. **Keyboard is first-class**：每个动作有快捷键，可被 ⌘K 触发。
4. **Dark-native**：暗色为原生默认，亮色是变体而非补丁。
5. **Motion with restraint**：动效用 Framer Motion，但只在"状态变化有意义"处出现，默认 ≤200ms。

## 3. 暗色模式实现

### 3.1 CSS 变量切换（无重渲染）

- 主题由 `<html data-theme="dark|light">` 控制，所有颜色走 CSS 变量，**不依赖 JS 主题对象重渲**。
- 默认 `dark`（暗色原生）；首访按系统 `prefers-color-scheme` 初始化，之后以用户选择持久化（localStorage `nc-theme`）。
- 切换只改 `data-theme` 属性，瞬时完成；不闪白（初始内联脚本在 `<head>` 同步设好属性）。

```css
/* tokens.css 结构 */
:root, [data-theme="dark"] {
  --bg-base: hsl(225 14% 9%);
  --text-primary: hsl(220 20% 92%);
  /* ...dark 全套 */
}
[data-theme="light"] {
  --bg-base: hsl(0 0% 100%);
  --text-primary: hsl(225 25% 14%);
  /* ...light 全套 */
}
```

### 3.2 切换组件与守卫

- `ThemeToggle` 仅切属性 + 写 localStorage，不触发全量 re-render。
- **强约束**：组件不得写 `color: #fff` 之类裸值；一律 `var(--text-primary)`。CI 用 stylelint 拦截裸色（见 §8）。
- 图片/插画：提供明暗两套或走 `filter` 适配（如 logo 用 `currentColor`）。

### 3.3 对比度基线

暗色正文对比度 ≥ 4.5:1（AA）；语义色在各自底上可读（通过 §9 a11y 校验）。切换主题后必须重新过对比度自检。

## 4. 组件库清单（Radix / shadcn 定制）

来源：02 §9（基于 Radix/shadcn 定制）、05 §2（`design/primitives`）。每个组件状态：默认 / hover / focus / disabled / loading。

> 统一约定：所有交互组件接受 `disabled`、`loading` 透传；focus 态统一 `--shadow-focus` 环；loading 用内联 spinner（≤400ms）或骨架。

### 4.1 Button
- **用途**：主操作。变体：`primary`(品牌填充) / `secondary`(中性描边) / `ghost`(无底) / `danger`(危险红) / `subtle`(弱)。
- **尺寸**：`sm`(28px) / `md`(32px) / `lg`(36px)，圆角 `--radius-md`。
- **状态**：
  - default：primary 用 `--brand-500` 底 + `--brand-foreground` 字
  - hover：primary → `--brand-600`
  - focus：2px `--brand-500` 环（offset 2px）
  - disabled：降透明度 40%，禁止指针
  - loading：左置 12px spinner，禁用文本点击

### 4.2 Input
- **用途**：单行文本输入。变体：`default` / `error`(危险红边) / `success`。
- **状态**：default `--bg-subtle` 底 + `--border-subtle`；hover `--border-strong`；focus `--shadow-focus`；disabled 灰底；loading 右侧 spinner。

### 4.3 Select
- **用途**：单选下拉（基于 Radix Select）。变体：`default` / `compact`(表格内)。
- **状态**：trigger 同 Input；面板 `--bg-elevated` + `--shadow-md` + `--z-popover`；option hover `--bg-muted`；选中项左侧勾 + 主色；disabled 灰。

### 4.4 Dialog
- **用途**：确认/表单弹窗（Radix Dialog）。圆角 `--radius-xl` + `--shadow-lg` + 遮罩 `--z-overlay`。
- **状态**：入场 `--dur-slow` + `--ease-out`（轻微 scale 0.98→1）；Esc/点遮罩关闭；focus trap 默认开（a11y）。

### 4.5 Toast
- **用途**：轻量通知（Radix Toast）。`--z-toast`。变体：`success`/`warning`/`danger`/`info`（语义色左边条）。
- **状态**：入场从右下滑入 `--dur-base`；停留 4s 自动消；hover 暂停计时；堆叠上限 3 条。

### 4.6 Progress
- **用途**：AI 任务/生成进度。变体：`linear`(细条) / `radial`(环形，面板用)。
- **状态**：default `--brand-500` 填充 + `--bg-muted` 轨道；indeterminate 用流光动画（≤1.4s 循环）；完成变 `--success`。

### 4.7 Tabs
- **用途**：面板内分栏（编辑器/属性）。下划线式（active 主色下划 + `--text-primary`），圆角用 `sm` 药丸变体可选。
- **状态**：default 次文字；hover 主文字；active 主色下划 + 主文字；focus 环；disabled 灰。

### 4.8 Tooltip
- **用途**：图标/缩写解释。`--z-tooltip` + `--shadow-md` + `--bg-elevated`。延迟 200ms 出现；暗色下浅底深字（反转对比保可读）。
- **状态**：入场 `--dur-fast` 淡入；hover 不消失；离开淡出。

### 4.9 DataTable
- **用途**：内容矩阵/知识库/指标表（高密度，学 Linear）。紧凑行高 32px，表头 `--text-secondary` 小字。
- **状态**：行 hover `--bg-muted`；选中行 `--brand-50` 底（暗色用 `--brand-700`/低透明）；排序头 hover 主色；空数据走 EmptyState（§4.11）；loading 行骨架。
- **特性**：列宽拖拽、虚拟滚动（>500 行）、行内 AI 操作入口。

### 4.10 EmptyState
- **用途**：无数据占位（列表/搜索/结果为空）。图标（线性，主色 300）+ 标题 + 一句说明 + 主操作按钮。
- **状态**：静态；主按钮可 hover/focus/disabled。

### 4.11 RadarChart
- **用途**：Reviewer 7 维质量分雷达（一致性/OOC/节奏/设定/前文/钩子/文学性）。基于 Recharts 定制主题色。
- **状态**：默认品牌填充 20% 透明 + 主色描边；hover 维度高亮 tooltip；对比模式叠加两期（实线 vs 虚线）。

### 4.12 DiffView
- **用途**：版本树 diff / AI 重写对比（Tiptap 文本 diff + 结构化字段 diff）。基于 diff-match-patch。
- **状态**：增行 `--success-bg` + 左绿条；删行 `--danger-bg` + 左红条；inline 改动黄底；hover 行高亮。支持"并排 / 行内"两种模式切换。

### 4.13 CommandK
- **用途**：全局命令面板（学 Raycast）。⌘K 唤起，`--z-popover` + `--shadow-lg`。模糊搜索所有动作（新建/运行工作流/跳转/AI 操作）。
- **状态**：input focus 环；结果项 hover/方向键选中 `--bg-muted`；分组标题 `--text-muted` 小字；空结果显示 EmptyState 内联；入场 `--dur-base` 上滑淡入。

### 4.14 其它 primitives
- `Switch` / `Checkbox` / `RadioGroup` / `Slider` / `Popover` / `DropdownMenu` / `Tooltip` / `Avatar` / `Badge` / `Skeleton` 均走 Radix + tokens 定制，状态规则同上（default/hover/focus/disabled/loading 中相关者）。

## 5. 布局规范

### 5.1 三栏工作台（主框架）

```
┌────────────┬──────────────────────────────┬──────────────┐
│ 项目树侧栏  │        主工作区               │   AI 面板     │
│ (220~260px) │   (编辑器 / 矩阵 / 看板)      │ (320~360px)  │
│ 可折叠      │   自适应填充                  │  可折叠/吸附  │
└────────────┴──────────────────────────────┴──────────────┘
       顶栏(48px): 面包屑 · 搜索 · ⌘K · 主题 · 头像
```

- **左栏**：项目树（书籍结构/知识/工作流）。折叠后留 48px 图标轨。
- **中栏**：主区，按页面切换；编辑器为沉浸态（可隐藏左右栏进入纯写作，Esc 退出）。
- **右栏**：AI 面板（当前章上下文/AI 操作/质量分/版本树）。非编辑页可隐藏。

### 5.2 响应式断点

| 断点 | 宽度 | 布局 |
|---|---|---|
| `sm` | ≥640px | 单栏（移动优先） |
| `md` | ≥768px | 双栏（左栏 + 主区） |
| `lg` | ≥1024px | 三栏（加 AI 面板） |
| `xl` | ≥1280px | 三栏 + 主区最大宽 1200px 居中 |
| `2xl` | ≥1536px | 全展开，主区放宽 |

### 5.3 移动端精简

- 默认单栏，左栏/右栏改为底部 Tab 或抽屉（Sheet）。
- 命令面板 ⌘K 在移动端用搜索图标触发（无物理键盘时显示）。
- 高密度表（DataTable）移动端降级为卡片列表；编辑器保留核心工具栏，高级操作收进菜单。
- 触控目标 ≥32px（比桌面 hover 态更大）。

## 6. 编辑器专属组件

来源：05 §2（`components/editor/` Tiptap 扩展）。

### 6.1 Tiptap AI 浮条（Floating AI Bar）
- 选中正文即浮现（--z-floating），含：续写 / 改写 / 润色 / 扩写 / 缩写 / 去 AI 味 / 风格化。
- 走 Gateway，同 Editor AgentSpec 计费（02 §7 去 AI 味等操作集）。
- 浮条入场 `--dur-base` 上浮淡入；失焦/无选中消失；暗色 `--bg-elevated` + `--shadow-md`。

### 6.2 行内批注（Inline Comment）
- 选区可加批注（评审意见/待改点），右侧 margin 显示气泡。
- 状态：未解决（主色点）/ 已解决（成功绿）/ 我发起（品牌描边）。
- 批注与版本绑定，恢复历史版本时批注随上下文显示。

### 6.3 版本树面板（Version Tree）
- 展示内容版本树（C5），AI 重写产生分支而非覆盖（02 §6）。
- 节点：manual / ai_rewrite / auto_save / restore 四色点区分。
- 操作：点击节点预览 diff（DiffView）、一键恢复（写新 restore 版本，非覆盖）。

### 6.4 Diff 视图（DiffView 复用）
- 版本间对比用 DiffView（§4.12）；AI 重写前后并排，接受/拒绝逐段。
- 三方对比（Offline L2 冲突，02 §9）：本地分支 vs 服务端 vs 共同祖先，人选择合并。

## 7. AI 无处不在的交互标准

来源：02 §9（AI 无处不在）。所有入口都是 Gateway 的薄调用，UI 成本为主。

### 7.1 全局 ⌘K（CommandK）
- 任何页面 ⌘K / Ctrl+K 唤起；命令含"AI：续写本章 / AI：生成社媒文案 / AI：分析热点"等。
- 命令可带上下文（当前选中/当前章节）自动注入。P1 优先级（列入 MVP）。

### 7.2 选中浮条（Floating Bar）
- 编辑器选中即出（§6.1）；非编辑器场景（如知识卡、社媒草稿）选中文本同样出精简浮条。

### 7.3 右键 AI 菜单（Context Menu）
- 正文/实体卡片右键，菜单含"AI 润色 / AI 扩写 / AI 解释 / AI 续写"。
- 基于 Radix ContextMenu，暗色 `--bg-elevated` + `--shadow-md`。

### 7.4 实体卡片 AI 补全按钮
- CharacterCard / 世界观卡 / 风格卡右上"AI 补全"按钮（呼应 02 §9 实体卡片 AI 补全）。
- 点击走对应 Agent（Character/Story Architect），结果 diff 预览后写入（经版本系统）。

### 7.5 发布页 AI 优化标题
- 发布页"AI 优化标题"按钮（02 §9），调 Social Media / Publisher Agent，给多平台备选标题 + SEO 标题。

### 7.6 交互底线
- 任何 AI 操作不得阻断编辑：结果以浮层/diff/面板呈现，可拒绝。
- 所有 AI 操作带 loading（Progress）+ 成本提示（若超预算预拦截）。
- AI 结果可一键"插入/替换/新建版本"，不做静默覆盖。

## 8. UI 验收五条规范

每个组件/页面合入前必须过以下五条（写进 PR 模板与视觉评审）：

1. **只用 tokens，禁裸色值**：所有颜色/间距/字号/圆角来自 §2 CSS 变量。stylelint 守卫（见下）拦截裸 `color`/`background`/`px` 字面量（允许 `0`、`inherit`、`transparent`、CSS 变量）。
2. **明暗双模式截图**：PR 须附 dark + light 两张关键态截图（含空/加载/错误态之一），对比度达标。
3. **空 / 加载 / 错误三态**：列表/面板/编辑器必须有 EmptyState（§4.10）、Skeleton/Progress（loading）、错误边界（danger 文案 + 重试）。
4. **键盘可达**：所有交互有焦点环（--shadow-focus）、可 Tab 抵达、有快捷键；Dialog/CommandK 有 focus trap；无键盘死路。
5. **与对标产品并排自评**：关键页（列表/编辑器/命令面板）与 Linear/Notion/Raycast 同屏截图自评信息密度与流畅度，差距记录进 issue。

### 8.1 stylelint 守卫配置（示例）

```js
// stylelint.config.js —— 拦截裸色值与非常规单位
module.exports = {
  rules: {
    "color-no-hex": true,            // 禁止裸 HEX，必须 var()
    "color-function-notation": "modern",
    "declaration-no-important": true,
    "length-zero-no-unit": true,
    "custom-property-pattern": "^[a-z][a-z0-9-]*$",  // token 命名
  },
  ignoreFiles: ["**/tokens.css"],     // tokens 定义文件豁免
};
```

> 例外：图形/图表库（Recharts）允许在其作用域用主题映射对象，但须从 tokens 取值，不得硬编码。

## 9. 无障碍（a11y）基线

### 9.1 对比度
- 正文文本 ≥ 4.5:1（AA）；大号文本 ≥ 3:1；语义色在底上可读（见 §3.3）。
- 暗色中性底已避开纯黑，保证正文对比；切换主题后重新自检。

### 9.2 焦点环（Focus Ring）
- 所有可聚焦元素：`:focus-visible` 显示 `--shadow-focus`（2px 主色环，offset 2px）。
- 禁止 `outline: none` 且无替代；鼠标点击不显环（`:focus-visible` 区分键鼠）。

```css
:focus-visible { outline: none; box-shadow: var(--shadow-focus); }
```

### 9.3 ARIA 与语义
- 用原生语义元素（button/input/nav/main）；非原生组件（Radix）自带 ARIA，不手改破坏。
- Dialog：`role="dialog"` + `aria-modal` + focus trap（Radix 默认）。
- DataTable：表头 `scope`、排序 `aria-sort`、行 `aria-selected`。
- Toast：`role="status"` / `aria-live="polite"`；错误用 `role="alert"`。
- 加载：`aria-busy` + spinner `aria-hidden`（避免读屏重复）。
- 图标按钮：`aria-label` 必填；纯装饰图标 `aria-hidden`。

### 9.4 动效与晕动
- 尊重 `prefers-reduced-motion: reduce`：关闭非必要动效（保留状态变化但去位移/淡入时长归 0）。

```css
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
}
```
