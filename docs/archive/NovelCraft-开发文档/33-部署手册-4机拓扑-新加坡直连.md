# 部署手册 — 4 机拓扑 / 新加坡直连（2026-07-13）

> 依据实测网络数据定案（详见对话）。本手册给出可直接执行的步骤，所有占位用 `<...>` 标出。

## 0. 角色分工（实测定案）

| 机器 | 角色 | 定案依据 |
|---|---|---|
| 🇸🇬 新加坡 43.156.17.78 | **NovelCraft 生产**（直连 + 本机 TLS，443 空闲） | 3.6G RAM（最大）+ 89G SSD + 3455 IOPS + 到 DeepSeek 最快 0.683s；唯一硬件够且 443 空的宿主 |
| 🇺🇸 美国 192.227.222.177 | WordPress（发布回执）+ 异地加密备份目的地 | 44G SSD + 最好 IOPS 4428 + 0 丢包；443 被 xray 占，需把 xray 挪到 8443 |
| 🇯🇵 日本 199.30.91.54 | 保持 frps（笔记本 Dev 用）；仅当新加坡网络变差时升级为 TLS 前置 | 72ms/0% 网络最好，但 1.9G/15G 当不了宿主 |
| 🏠 ASUS 笔记本 | **仅 Dev**（经日本 frp 访问） | 磁盘随机 **101 IOPS / 0.4 MB/s / 96.9% 利用率** → Postgres 会被拖死，生产出局 |

**关键实测**：DeepSeek 四台耗时几乎相同（0.68–0.89s），非选型因素；新加坡直连 RTT ~255ms、丢包 3.3%（对 1–5 人写作工具可接受，瓶颈是 DeepSeek 生成而非网络 RTT）。

## 1. 全部 VPS：开启 BBR（改善有损链路）

```bash
echo 'net.core.default_qdisc=fq' | sudo tee -a /etc/sysctl.conf
echo 'net.ipv4.tcp_congestion_control=bbr' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
lsmod | grep bbr        # 有输出即生效
```

## 2. 新加坡：装依赖 + 内存兜底

```bash
sudo apt update && sudo apt install -y docker.io docker-compose-plugin nginx certbot
# 4G swap（3.6G 内存 + Postgres 必须留兜底）
sudo fallocate -l 4G /swapfile && sudo chmod 600 /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
```

确认端口：`sudo ss -tlnp | grep -E ':80|:443'`。应看到 xray 占 80/8080/8443，**443 空**。

## 3. 新加坡：拉代码 + 配置

```bash
git clone https://github.com/Catatlina/NovelCraft-Personal-Studio.git && cd NovelCraft-Personal-Studio
cp .env.production.example .env
# 按注释填 .env：JWT/Fernet/CORS(=https://app.你的域名)/POSTGRES_PASSWORD/DEEPSEEK_API_KEY
#   openssl rand -hex 32                                              # JWT
#   python3 -c "from cryptography.fernet import Fernet;print(Fernet.generate_key().decode())"  # Fernet
```

## 4. 新加坡：证书（因 80 被 xray 占，用 DNS-01）

```bash
# 先把 app.你的域名 A 记录指向 43.156.17.78
sudo certbot certonly --manual --preferred-challenges dns -d app.你的域名
# 按提示在域名处加一条 _acme-challenge TXT 记录，验证通过即签发
# （若你的 DNS 是 Cloudflare/阿里云等有 API 的，装对应 certbot 插件可自动续签）
```
> 若某天愿意腾出 80：`certbot --nginx` 就能全自动续签，更省心。

## 5. 新加坡：启动 + 挂 TLS

```bash
# 启动应用（前端绑 127.0.0.1:8090，避开 xray 的 80；worker 单进程；跳过 flower）
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build --scale flower=0
docker compose ps         # api/worker/postgres/redis/frontend 应为 healthy

# 挂 TLS：把模板里的 app.example.com 改成你的域名
sudo cp nginx/novelcraft-singapore.conf /etc/nginx/sites-available/novelcraft
sudo sed -i 's/app.example.com/app.你的域名/g' /etc/nginx/sites-available/novelcraft
sudo ln -sf /etc/nginx/sites-available/novelcraft /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

**验收**：浏览器开 `https://app.你的域名` 绿锁、能注册登录；`curl https://app.你的域名/api/v1/healthz` 返回 db/redis ok。

## 6. 美国：WordPress（发布回执验收）

```bash
# 先把 xray 从 443 挪到 8443（美国 443 被占），腾出 443
# blog.你的域名 A 记录 → 192.227.222.177
sudo certbot --nginx -d blog.你的域名          # 美国 80 空闲，HTTP-01 直接可用
# docker 跑 WordPress（wordpress + mariadb），nginx 反代到它
```
注册平台账号（凭据经 Fernet 加密落库，响应不回显）：
```bash
curl -X POST https://app.你的域名/api/v1/publish/account/register \
  -H "Authorization: Bearer <token>" -H "X-CSRF-Token: <csrf>" -H "Content-Type: application/json" \
  -d '{"platform":"wordpress","account_name":"blog",
       "credentials":{"wp_url":"https://blog.你的域名","wp_user":"admin","wp_pass":"<应用密码>"}}'
```
发布一篇 → 查 `publish_records` 看到真实 `draft_created` + WP 文章 URL = 整链通过。
> Medium 自 2023 起基本不再发新 Integration Token，优先用 WordPress 验收。

## 7. 监控（不占服务器）

- **Telegram**：`@BotFather` `/newbot` 拿 token；给 bot 发消息后开 `https://api.telegram.org/bot<token>/getUpdates` 取 `chat.id`；填 `.env` 的 `TELEGRAM_BOT_TOKEN`/`TELEGRAM_CHAT_ID`，重启 worker：`docker compose ... up -d worker`。验收：把某项目预算调到极低跑一次生成，应收到告警。
- **Sentry**：[sentry.io](https://sentry.io) 免费额度拿 DSN → 填 `.env` `SENTRY_DSN`。低配 VPS 别自托管 GlitchTip。
- **Uptime**：日本或美国上 cron：`*/5 * * * * curl -fsS https://app.你的域名/api/v1/healthz >/dev/null || <curl Telegram 告警>`。

## 8. 异地备份（compose 已每日 pg_dump，推到美国）

```bash
# 新加坡装 rclone，配置到美国 VPS 的 sftp remote，每日同步 backups/
0 4 * * * rclone sync /path/NovelCraft-Personal-Studio/backups us-vps:/backups/novelcraft
```
满足 RPO≤24h；恢复演练：在美国把某个 `.sql.gz` 拉回、`gunzip | psql` 到临时库核对。

## 9. 可选升级：日本 TLS 前置（仅当新加坡网络对你日常用变得难以忍受）

- 域名改指日本 199.30.91.54；日本 nginx 443 终结 TLS，反代到新加坡。
- **必须**在日本↔新加坡之间加 **WireGuard 隧道**，让登录令牌不明文过公网；日本 nginx `proxy_pass` 指向新加坡的 WireGuard 内网 IP。
- 这是叠加改动，不需重建应用。需要时我再给完整 WireGuard + 日本 nginx 配置。
