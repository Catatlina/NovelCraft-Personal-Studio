# AI 模型系统设计文档 (Enterprise Edition)

## AI Gateway 架构

统一入口 `gateway.complete()` → 预算检查 → 路由 → Prompt 渲染 → Provider 调用 → Schema 校验 → 追踪落库

## 支持 Provider

| Provider | 模型 | 适用场景 |
|---|---|---|
| DeepSeek | deepseek-chat | 草稿/摘要/字段生成 |
| Claude | claude-sonnet-4 | 关键章节/风格化/审核 |
| OpenAI | gpt-4o | 通用/复杂推理 |
| Gemini | gemini-2.0-pro | 长上下文备胎（1M token） |

## 模型路由

```sql
model_routes(task_type, primary_provider, primary_model, fallbacks JSONB, params JSONB)
```

路由配置热更新（Redis 缓存），支持百分比分流 A/B 测试。

## 降级策略

主模型失败 → 备用1 → 备用2 → PENDING_PROVIDER → Telegram 告警

## 预算控制

三级熔断：单任务 / 单项目 / 单日
Redis 计数 + 落库对账，超限置 PENDING_BUDGET

## 全量追踪

ai_calls 表记录九要素：
provider / model / prompt_name / prompt_version / input / output / prompt_tokens / completion_tokens / cost / latency_ms / status

支撑成本统计、模型对比、A/B 分析、Prompt 实验室回放。
