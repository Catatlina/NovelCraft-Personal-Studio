# 小说出海系统设计 (Enterprise Edition)

## 概述

将中文网络小说翻译并分发到全球主流网文平台的全链路系统。

## 出海全链路

```
中文原文 → 术语提取 → 分段翻译 → 文学化改写 → 文化本地化
  → 禁忌检查 → SEO优化 → 格式适配 → 平台发布 → 数据回流
```

## 翻译管线

### 1. 术语管理
- 自动提取人名/地名/功法/专有名词
- 术语库统一翻译
- 上下文一致性检查

### 2. 分段翻译
- 章节级拆分，保持上下文连贯
- 支持多模型（Claude 文学翻译 / GPT 通用 / DeepSeek 经济）

### 3. 文学化改写
- 直译 → 文学化
- 对话自然化
- 中文梗/俗语 → 英语等价表达

### 4. 文化本地化
- 人名地名适配
- 计量单位转换
- 文化概念解释
- 目标市场禁忌检查

## 海外平台

| 平台 | 类型 | 收入模式 |
|---|---|---|
| Royal Road | Web 小说 | 读者打赏/广告 |
| WebNovel | 网文平台 | 章节付费/签约 |
| Scribble Hub | 网文社区 | 读者互动/打赏 |
| Amazon KDP | 电子书/实体 | 销售分成/KU |
| Medium | 长文平台 | 会员分成 |
| Substack | Newsletter | 付费订阅 |

## KDP 发布流程

1. 内容转换（Tiptap JSON → EPUB/MOBI）
2. 封面设计（AI 生成 + 人工选择）
3. 元数据优化（标题/副标题/分类/关键词/描述）
4. 定价策略（竞品分析 + AI 建议）
5. A+ 内容生成（图文详情页）

## 数据回流

- 各平台数据自动/手动采集
- 阅读量/评分/评论/收益/排名
- 跨平台对比分析
- ROI 计算（翻译成本 vs 出海收益）
