# 后端代码工程规范 (Enterprise Edition)

> 企业版后端工程规范，基于 FastAPI + Python 3.11+。

## 项目结构

```
backend/
├── app/
│   ├── api/v1/           # 版本化路由
│   ├── services/         # 业务逻辑层
│   ├── repositories/     # 数据访问层
│   ├── models/           # SQLAlchemy 模型
│   ├── schemas/          # Pydantic 请求/响应 schema
│   ├── ai/               # AI Gateway
│   │   ├── gateway.py
│   │   ├── budget.py
│   │   ├── router.py
│   │   └── providers/    # DeepSeek/Claude/OpenAI/Gemini
│   ├── agents/           # 10 Agent 定义
│   ├── workers/          # Celery 任务 + beat
│   ├── core/             # 配置/安全/中间件
│   └── tests/            # pytest
├── alembic/              # 数据库迁移
├── docker-compose.yml
└── Dockerfile
```

## 编码规范

### 三条铁律
1. **LLM 调用只准走 Gateway** — CI grep 守卫
2. **写操作只准走 VersionedRepository** — 先快照后更新
3. **内容读写只准走 content_service** — meta Schema 校验

### 代码风格
- Python 3.11+，类型注解全覆盖（mypy strict）
- 异步优先（async/await），阻塞操作放 Celery
- Pydantic v2 做所有输入输出校验
- structlog 结构化日志
- Alembic 线性单头迁移
- pytest + pytest-asyncio（覆盖率 ≥ 80%）

### 命名规范
- 文件名：snake_case（content_service.py）
- 类名：PascalCase（ContentService）
- 函数/变量：snake_case（get_content_by_id）
- 常量：UPPER_SNAKE_CASE（MAX_RETRY_COUNT）

### 错误处理
- 统一异常处理中间件
- 错误码规范：`{模块}_{错误类型}`
- AI 调用失败分级：重试 → 降级 → PENDING_PROVIDER → 告警

### Git 规范
- 分支：feature/ fix/ chore/
- Commit：conventional commits
- PR 必须通过 lint + type + test 方可合入
