# 项目管理开发计划 (Enterprise Edition)

## 开发方法论

- **里程碑制**：M(n) 未验收不开 M(n+1)
- **文档驱动**：代码 PR 与文档 PR 同评审
- **单人优先**：每周 15~25 小时，串行推进
- **门禁红线**：任何跨里程碑抢跑功能一律拒绝

## 开发周期

全量交付约 9~10 个月（40 周），52 个任务。

| 里程碑 | 周数 | 任务数 | 每周工时 |
|---|---|---|---|
| M1 地基+MVP | 1~5 | 14 | 15~25h |
| M2 小说引擎 | 6~16 | 16 | 15~25h |
| M3 内容工作室 | 17~26 | 10 | 15~25h |
| M4 发布出海 | 27~34 | 6 | 15~25h |
| M5 协作离线 | 35~40 | 6 | 15~25h |

## 风险管理

| 风险 | 等级 | 对策 |
|---|---|---|
| 全量交付周期长 | 高 | 严格里程碑制，每阶段可独立使用 |
| 自动发布封号 | 高 | 三模式，国内全自动默认关闭 |
| 热点数据源脆弱 | 中 | 可插拔 adapter，失败降级手动 |
| UI 第一 vs 带宽 | 中 | 设计系统先行，页面分级投入 |
| 成本失控 | 中 | 预算熔断从 MVP 内建 |

## 沟通与协作

- GitHub Issues 追踪任务
- PR Review 为必要流程
- 每周进度同步（文档 + Demo）
- 架构决策写 ADR（架构决策记录）

## 文档纪律

- 本套文档为唯一基线
- 代码 PR 与文档 PR 同评审
- 禁止只有标题的空壳文档
- 文档更新纳入 CI（link check + 格式校验）
