# 爆款分析系统设计 (Enterprise Edition)

## 概述

AI 驱动的爆款内容分析引擎，帮助创作者理解"为什么火"并应用到自己的创作中。

## 分析维度

| 维度 | 内容 | 产出 |
|---|---|---|
| 开篇结构 | 前 500 字的节奏/钩子/信息密度 | 开篇模板 |
| 爽点分析 | 爽点类型/密度/分布 | 爽点图谱 |
| 节奏分析 | 章内节奏/卷级节奏曲线 | 节奏模型 |
| 标签分析 | 热门标签/标签组合/趋势 | 标签策略 |
| 传播潜力 | 社交传播因子/话题性/争议性 | 传播评分 |
| 平台适配 | 各平台的爆款特征差异 | 平台策略 |

## 数据源

- 起点中文网（畅销/月票/推荐榜）
- 番茄小说（热门/新书榜）
- 小红书（爆款笔记分析）
- 公众号（10w+ 文章分析）
- 知乎（高赞回答分析）

## 分析工作流

```
选择目标榜单 → 抓取 top N → 逐篇深度分析 → 汇总趋势报告
  → 提取可复用模式 → 入库（Knowledge Hub）→ 注入生成上下文
```

## 应用场景

- **选题阶段**：分析当前热门题材/标签 → 选题推荐
- **开篇阶段**：应用爆款开篇结构模板
- **连载中**：监控节奏是否偏离爆款曲线
- **复盘阶段**：对比自己的书与爆款的差距

## 扫榜自动化

- 定时扫榜（每日/每周）
- 自动生成扫榜报告
- 趋势变化告警（某题材突然爆发）
