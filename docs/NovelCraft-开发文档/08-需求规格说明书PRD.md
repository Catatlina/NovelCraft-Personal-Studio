## 1. 产品概述

### 1.1 愿景与使命
- **愿景**：让单个创作者具备一支专业内容团队的产能——从选题、创作、审核、发布到数据反馈、持续优化的完整闭环，由一人 + AI 完成。
- **使命**：把"功能一个不少、架构保持轻量"变成工程现实：以 8 个平台能力为骨架，以配置态承载长尾功能。

### 1.2 产品原则（10 条，需求取舍的最高准则）
1. **UI 第一**（对标 Cursor/Notion/Linear/Raycast）；
2. **Everything is Content**（统一内容模型）；
3. **Everything is Knowledge**（统一知识入库 + RAG）；
4. **Everything is Traceable**（AI 调用全记录）；
5. **Everything is Versioned**（一切可版本、可回滚）；
6. **Automation First**（工作流引擎驱动一切自动化）；
7. **Offline First**（分级离线，L1~L3）；
8. **AI 无处不在**（选中即用、⌘K、右键 AI）；
9. **叙事一致性护城河**（百万字不崩）；
10. **轻量自托管**（单 VPS，无微服务/K8s/多租户）。

### 1.3 范围边界（明确不做）
微服务、K8s、SaaS 多租户、企业组织架构、支付计费、复杂权限树、CRDT 实时协同、离线自动合并、全功能离线。

### 1.4 需求收敛模型：250+ 功能 = 8 平台能力 + 配置态
| 能力 | 名称 | 承载的功能类别 | 清单占比 |
|---|---|---|---|
| C1 | 统一内容模型 | 全部体裁（长/短篇、微小说、公众号、小红书、知乎、短视频脚本…） | ~25% |
| C2 | 工作流引擎 | 自动连载、批量、定时、一稿多平台、暂停/恢复/取消/重试、人工确认、预设+自定义 | ~15% |
| C3 | AI Gateway + Agent | 10 Agent、4 家模型路由/降级/重试、Token 预算、成本、模型对比、A/B、Prompt 实验室 | ~15% |
| C4 | Knowledge Hub | 素材中心、RAG、四库、项目+全局知识库、风格学习、拆书分析 | ~12% |
| C5 | 通用版本系统 | 章节/内容/人物/世界观/Prompt/知识版本、对比、恢复 | ~8% |
| C6 | 叙事一致性引擎 | 伏笔、时间线、人物弧线、OOC、设定冲突、防崩、7 层上下文、分层摘要 | ~12% |
| C7 | 发布网关 + 数据回流 | 平台账号、定时/自动发布、格式适配、发布记录、数据回流、表现分析、ROI | ~8% |
| C8 | 追踪与治理 | AI 调用全记录、审计日志、操作日志 | ~5% |

**新增一种体裁 ≠ 新模块**：= C1 一个 content type + C2 一个工作流模板 + C3 一个 prompt。**平台能力做厚，具体功能做薄。**

## 2. 用户角色与画像

| 角色 | 描述 | 核心诉求 | 权限 |
|---|---|---|---|
| 独立创作者（主画像） | 一人身兼选题/写作/编辑/运营 | 用最少精力产出高质量、可持续更新的内容 | Owner |
| 网文连载作者 | 追求百万字设定不崩、稳定日更 | 叙事一致性 + 自动连载 + 防崩巡检 | Owner |
| 自媒体矩阵运营者 | 一稿多平台分发、追热点 | 一稿多平台 fan-out + 热点晨报 + 数据回流 ROI | Owner |
| 小团队协作成员 | 2~5 人协作（主编/编辑/审校） | 分工协作、修改留痕、不越权 | Editor / Viewer |
| 出海作者 | 中文作品翻译分发海外平台 | 翻译/文学化/本地化/禁忌检查/海外平台适配 | Owner |

**协作角色权限（1~5 人极简 RBAC）**：
- **Owner**：全权（含删项目、管成员、改预算/模型路由）；
- **Editor**：读写内容、跑工作流；**不可**删项目/管成员/改预算；
- **Viewer**：只读。

## 4. 典型用户故事（关键闭环）

- **UC-01（MVP 核心）**：作为独立创作者，我输入一句话灵感（+ 题材/风格/目标字数），系统在 15 分钟内自动产出书名（3 候选）→ 简介卖点 → 世界观 → 人物 → 总纲 → 第一章 → 7 维审核，我在审阅页选定标题、确认或退回重跑单步，然后在编辑器里选中润色/改写/续写并保存（自动版本快照）。
- **UC-02（护城河）**：作为网文作者，我开启自动连载，系统按 7 层上下文 + 实体状态 + 伏笔提醒稳定生成，每 10 章自动巡检，卷末需我人工复盘门禁通过才继续；30 万字后人工抽检一致性错误率 <5%。
- **UC-03（矩阵运营）**：作为自媒体运营者，我把一章小说一键 fan-out 到 5 个平台（公众号/头条/知乎/小红书/视频脚本），5 分钟内生成且血缘可查；每天早晨"热点晨报"工作流自动抓热点→生成三平台稿→停在待我审核。
- **UC-04（发布回流）**：作为运营者，我把已审内容半自动发布（Playwright 填充，我点最终发布），数据回流后在 ROI 面板看到每篇的成本-收益对照。
- **UC-05（出海）**：作为出海作者，我跑出海工作流，产出英文稿并通过禁忌检查，适配 Royal Road/KDP 格式后发布。
- **UC-06（协作）**：作为主编，我邀请一位 Editor 协作编辑，其无法删项目/改预算，所有修改留痕可回滚。
- **UC-07（离线）**：作为通勤中的作者，断网时我能浏览/搜索项目并编辑暂存、排队 AI 任务，恢复网络后草稿自动提交、AI 任务自动执行；若他人已改同章，则弹三方对比让我选择。

## 5. 验收标准（按里程碑，与路线图/测试计划对齐）

### M1 地基 + MVP（P0）门禁
1. 新用户 ≤15 分钟灵感→过审第一章，全程无需看日志；
2. 杀 worker/断网后 run 断点续跑；human 节点挂起 24h 后确认仍可继续；
3. 单次 bootstrap 成本 ≤¥2 预算，超限熔断为 PENDING_BUDGET 非失败；
4. 任意保存/AI 重写可版本恢复；ai_calls 每次调用记录九要素完整；
5. V1 数据迁移校验通过且可回滚（**V1 收口 106 测试全绿为迁移前置门禁**）；
6. 单测总 ≥135 全绿；lint/type/build 过；三轮自审报告归档；
7. 备份恢复演练 1 次通过；失败告警实际收到 1 条测试通知。

### M2 小说引擎完全体（P1）
- 一本书连续生成 30 万字，人工抽检一致性错误率 <5%，7 维均分 ≥80；
- 伏笔到期回收提醒与校验生效；
- 自定义工作流可从零编排"细纲→正文→审核→返工"链并成功执行；
- 4 家模型路由与降级演练通过。

### M3 内容工作室（P2）
- 一章小说 5 分钟派生 5 平台格式且血缘可查；
- 每日晨报工作流连续 7 天无人值守成功（human 节点等待审核）；
- 知识库对 100+ 素材检索相关性抽检通过；
- 仿写相似度红线（≥0.75 强制重写）拦截生效。

### M4 发布·数据·出海（P3-A）
- 一篇内容从生成到（半自动）发布到数据回流全链可查；
- 国内平台未开全自动时只能停在半自动待点击态；开启后首篇仍出现 human 确认；
- 出海工作流产出英文稿并通过禁忌检查；ROI 面板展示成本-收益对照。

### M5 协作·离线·多端（P3-B）
- 第二账号以 Editor 加入协作编辑无越权；
- 断网可浏览/搜索/编辑并排队 AI 任务，恢复后自动执行；并发冲突弹三方对比、无静默覆盖；
- 手机浏览器可完成"审核确认 + 轻量编辑"；
- 全功能对照清单逐项勾验 250+ 功能映射通过。

## 6. 非功能需求（NFR）

| 编号 | 类别 | 需求 |
|---|---|---|
| NFR-01 | 性能 | SSE 进度延迟 ≤2s；单章生成端到端可接受；向量检索相关性抽检达标；30 万字连载压测通过 |
| NFR-02 | 可靠性 | 工作流断点续跑、节点幂等；备份 RPO≤24h、RTO≤1h；provider 全降级不丢任务（PENDING 恢复重试） |
| NFR-03 | 可扩展 | 新体裁/新平台仅需 type + meta schema + prompt + adapter，零核心表变更；垂直扩容优先 |
| NFR-04 | 安全 | JWT 双令牌轮换、Fernet 凭据加密、SQL 注入/XSS/CSRF/SSRF 防护、Prompt 注入防护、越权全拦截（详见 17 安全设计） |

| NFR-06 | 成本 | 三级预算熔断；max_tokens=目标字数×1.4；稳定上下文启用 prompt caching；成本可按章/篇/模型归集 |
| NFR-07 | 可维护 | 单 VPS + Compose，无微服务；structlog JSON 日志；1~5 人可维护 |
| NFR-08 | 可用性/UI | UI 第一五条验收（仅用 tokens、明暗双模式、三态齐全、键盘可达、对标自评）；a11y 对比度达标 |
| NFR-09 | 可追踪 | 全部 AI 调用与敏感操作可回放、可审计 |
| NFR-10 | 可移植 | Docker 镜像 tag 发布，make deploy 一键升级 + 失败自动回滚 |
