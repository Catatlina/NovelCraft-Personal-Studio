### 1.1 测试金字塔

以「真实依赖」为底座，严禁用假数据库/假队列裁剪链路。金字塔自底向上：

```
        /\
       /  \         UI 验收（少量，人工 + Puppeteer 关键路径）
      /----\        E2E 端到端（TestClient × 真 pg/redis，核心链路）
     /------\       集成测试（service/repository × 真 pg/redis/celery）
    /--------\      契约测试（OpenAPI schema + prompt 输出 schema）
   /----------\     单元测试（纯函数、AgentSpec 渲染、预算计数、diff）
  /------------\
 单元 > 集成 > E2E，安全/性能为横切层
```

- 单元：占比 ~55%，运行快、无 IO，覆盖纯逻辑（预算计算、版本 diff、相似度算法、DAG 拓扑）。
- 集成：占比 ~30%，**真实 pg/redis**，覆盖 repository/service 层与 Celery 任务落库。
- E2E：占比 ~10%，**TestClient × 真 pg/redis + 真 Celery worker（测试 compose）**，覆盖 bootstrap 全八节点、发布全链、离线重放。
- 横切：安全测试、性能压测、UI 验收，按里程碑触发。

### 1.2 覆盖率目标

| 层 | 目标 | 工具 | 红线 |
|---|---|---|---|
| 后端行覆盖 | ≥80% | pytest-cov | 门禁 <80% 拒合 |
| 后端分支覆盖 | ≥70% | pytest-cov | 工作流引擎/网关/版本系统分支必须全覆盖 |
| 前端组件覆盖 | ≥70% | vitest + @testing-library | 编辑器/审阅页/进度页 |
| Agent golden case | 10/10 Agent 各 ≥1 | pytest | CI 全绿 |
| 关键路径 E2E | 100% 覆盖 C1~C8 红线 | pytest E2E | 任一红线无 E2E 用例 = 门禁失败 |

### 1.3 测试铁律（强制）

1. **真实依赖**：PostgreSQL + pgvector、Redis、Celery worker 均通过 `docker-compose.test.yml` 拉起，**禁止用 sqlite/内存 dict/fakeredis/ mock 替代数据库与队列**。
2. **TestClient 真链路**：E2E 用 FastAPI `TestClient`（或 httpx + ASGITransport），经完整路由 → 依赖注入 → 中间件 → service → repository → 真 pg；Celery 用 `CELERY_TASK_ALWAYS_EAGER=False` + 真 broker 跑，或 `eager` 但保证落库路径一致（需注释说明）。
3. **外部 LLM 可 mock**：Provider 调用是唯一允许 mock 的边界，采用 **VCR 式录制回放**（record/replay），golden case 用录制快照断言 schema + 关键字段；禁止 mock 任何 DB/队列/内部的落库行为。
4. **全绿才打包**：lint/type/unit/integration/build 全绿方允许 `docker build` 与部署（07-规范 §3）。
5. **失败路径必测**：每个新功能必须带失败路径用例（预算超限、schema 失败、节点崩溃、冲突提交）。

### 1.4 CI 门禁（来源：07-工程协作规范 §7）

```
push / PR:
  ├─ ruff            (Python lint，0 error)
  ├─ mypy --strict   (后端类型，0 error)
  ├─ tsc --noEmit    (前端类型，0 error)
  ├─ stylelint       (design tokens 守卫：禁裸色值/魔法间距)
  ├─ 单测            (pytest × 真 pg/redis 服务容器，覆盖率门禁)
  ├─ prompt golden case 回归  (10 Agent × ≥1 case，assert schema+关键字段)
  ├─ LLM 直连 grep 守卫   (扫描是否误将 test mock 注入生产 prompt 路径)
  └─ docker build    (镜像可构建)
merge main:
  └─ tag → 构建镜像 → make deploy（备份→migrate→up→healthcheck→失败回滚）
```

`LLM 直连 grep 守卫` 示例（CI 脚本）：

```bash
# .github/ci/llm_guard.sh
# 禁止在 production 代码路径出现 mock provider 回退
if grep -rnE "MOCK_LLM|fake_provider|FakeCompletion" app/gateway/ app/agents/ ; then
  echo "ERROR: mock LLM 泄漏到生产 gateway/agents 路径"
  exit 1
fi
# 但测试目录允许
if grep -rnE "MOCK_LLM" app/ tests/ | grep -v "tests/"; then
  echo "ERROR: mock 仅允许出现在 tests/"
  exit 1
fi
```

| 类型 | 目标 | 关键工具 | 触发 |

### 3.3 外部 LLM 的 mock / 录制回放策略

**铁律边界**：LLM Provider 是唯一允许 mock 的边界；DB/队列绝不可 mock。

采用 VCR 式 `record/replay`：

```python
# tests/llm_replay.py
import vcr, json
import pytest

@pytest.fixture
def llm_cassette():
    # 首次运行录制真 LLM 响应到 cassettes/<case>.yaml
    # 后续运行回放，保证 golden case 稳定可断言
    with vcr.use_cassette(
        "tests/cassettes/bootstrap_gen_titles.yaml",
        record_mode="once",          # 已存在则回放，不直连
        match_on=["method", "scheme", "host", "path", "body"],
        filter_headers=["authorization"],
    ):
        yield

def test_bootstrap_gen_titles_schema(llm_cassette, db_session, owner_user):
    # 断言：输出 schema 合法 + 关键字段（title_candidates 长度 3）
    ...
```

- 录制产物提交 `tests/cassettes/`，作为 golden case 基线；
- CI 默认 `record_mode=none`（纯回放），仅本地手动 `record` 刷新；
- 任何 `app/gateway`、`app/agents` 内的 mock 回退均被 §1.4 grep 守卫拦截。

## 7. 性能压测方案

### 7.1 30 万字连载压测

目标：M2 验收「一本书连续生成 30 万字，人工抽检一致性错误率 <5%」。

```yaml
# tests/perf/serial_300k.yml
场景: 自动连载压测
数据: 预置 novel + 分卷大纲 + 人物状态表
步骤:
  - 触发自动连载 workflow（schedules），连续生成 300 章 × ~1000 字
  - 每章触发 7 层装配 + Reviewer 审核 + 版本快照
  - 监控: 单章生成 P95 延迟 / pg 连接数 / Redis 内存 / Celery 队列积压
通过线:
  - 全程无 worker 崩溃 / 无数据丢失
  - 一致性抽检错误率 <5%
  - 7 维均分 ≥80
  - 预算累计在配置上限内（熔断生效不雪崩）
```

### 7.2 并发与 SSE 延迟

| 指标 | 目标 | 工具 | 通过线 |
|---|---|---|---|
| SSE 进度推送延迟 | ≤2s | k6 + 自定义探针 | P95 ≤2s |
| 并发 bootstrap 数 | ≥5 并行 run | locust | 无 5xx；队列有序 |
| pg 向量检索 P95 | ≤500ms | pgbench + 召回脚本 | k=10 召回相关 |
| Redis 预算计数吞吐 | ≥1k ops/s | redis-benchmark | 计数准确无竞态 |

### 7.3 向量检索相关性抽检

```python
# tests/perf/vector_recall.py
# 对 kb_100 做抽样，人工标注 top-5 相关项，自动比对召回
def test_recall_quality():
    queries = load_sample_queries()        # 含人工 gold 标注
    for q in queries:
        hits = kb.search(q.text, kinds=q.kinds, k=5)
        assert recall_at_5(hits, q.gold) >= 0.8   # 抽检通过线
```
