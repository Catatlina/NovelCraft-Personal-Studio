# NovelCraft 新增需求任务分解（AI 可执行版）

> 日期：2026-07-13
> 执行原则：每次开发前必须先阅读并遵守《23-AI开发边界与交付真实性规范》；按序开发，前一任务未通过验收不得进入下一任务；所有 AI 能力必须走真实 provider，失败直接报错，不允许 mock、本地伪造、静默兜底。

## 全局硬约束

0. 开发前必读：任何 AI/人工开发者开始本文件任务前，必须阅读 `23-AI开发边界与交付真实性规范.md`、`PROJECT_PROGRESS.md`、最新审计报告；必须执行《23》§10 反撒谎扫描；未满足时不得宣称完成。
1. 真实 AI：所有创作、改写、审查、规划、仿写、自媒体生成必须调用 `app.gateway.complete()` 或 `complete_stream()` 的真实 provider 路径。
2. 失败语义：provider key 缺失、网络失败、模型返回非法 JSON、预算不足，必须向调用方返回明确错误；不得返回估算结果、术语替换结果、固定模板结果。
3. 自审要求：每个任务完成后必须运行对应后端测试、前端构建或端到端冒烟；涉及 AI 的功能需至少验证无 key 时明确失败，有 key 时可运行真实 provider smoke。
4. 文档要求：开发完成后更新本文件状态和相关接口说明；旧审计/废弃进度表只保留在 `docs/archive/`，不得再作为完成率依据。

## T1 去 Mock 与真实 AI 闭环

目标：彻底移除业务运行时 mock/fallback 内容生成。

状态（2026-07-14）：已完成代码整改并完成真实 Provider 冒烟。`mock` provider 不再作为产品能力；Provider 失败在同步/API 和批次关键路径进入明确错误/`failed`，不产出占位内容。

开发项：

- 网关不接受 `mock` provider，不读取 `fallback_json` 生成替代内容。
- 配置 API 禁止保存 `mock` provider，fallback 列表置空。
- 出海翻译、分层大纲、热点生成、编辑器审查等链路禁止本地估算兜底。
- 保留单元测试对外部网络的 monkeypatch，但测试不得把 mock 路由当作产品能力。

验收：

- `rg "_mock_output|provider == \"mock\"|NOVELCRAFT_ALLOW_MOCK|fallback_term_only|estimated structure" backend/app` 无业务生成路径命中。
- 无 API key 时，AI 相关接口返回明确 4xx/5xx 错误。
- 有真实 key 时，至少跑通一个编辑器改写、一个热点生成、一个 bootstrap 节点。

## T2 书库列表与详情

目标：书库按书籍创建顺序展示，最新创建为 1，并提供详情页。

状态（2026-07-14）：已完成后端列表/详情接口与前端详情页接线；待浏览器 E2E 做最终视觉验收。

开发项：

- `GET /api/v1/ranking/library/books` 按 `created_at DESC` 返回：书名、部分简介、类型、创建时间、最新章节、章节数、字数。
- `GET /api/v1/ranking/library/books/{book_id}` 返回：书名、简介、大纲、最新章节、全部章节、知识条目。
- 前端书库默认显示精简列表，点击“查看详情”进入下级详情页。

验收：

- 新建两本书后，后创建的书显示序号 1。
- 列表不展示冗余工作流按钮为主信息；按钮只作为操作区。
- 详情页能看到全部章节，空章节时显示明确空状态。

## T3 自媒体热点采集与生成

目标：自动采集热点，并能按热点生成多平台内容。

状态（2026-07-14）：已完成采集源扩展、逐源失败状态、热点生成接口和前端按钮；小红书/抖音/X 等需真实账号、cookie、token、代理或人工源地址的能力，已接入“设置 → 平台连接”可视化配置，未配置或调用失败时明确报错，不伪造数据。新增历史归档回填：配置 `history_url`（支持 `{date}`）后可回填近 7 天真实历史快照；无授权历史源时返回 unsupported/502，不用当前热点伪造过去 7 天。

开发项：

- 热点源覆盖百度、知乎、微博；小红书、抖音、X 支持 `HOTSPOT_*_URL` 或 cookie 配置，未配置时报错，不伪造数据。
- 历史热点支持 `HOTSPOT_*_HISTORY_URL` 或“平台连接 → 历史归档 URL”，按日期回填；未配置历史源时必须明确 unsupported，不得合成历史。
- `POST /api/v1/hotspots/generate` 根据热点生成公众号、头条号、百家号、大鱼号、小红书、抖音草稿。
- 前端热点卡片提供“按此热点生成”按钮。
- `GET/POST/DELETE /api/v1/platform-connections` 提供真实平台账号/API/人工配置的可视化填写入口；敏感字段 Fernet 加密保存，列表只返回配置状态和缺失字段，不回显密钥。

验收：

- 单源失败不会伪造热点，接口返回 `sources` 状态。
- 全源失败返回 502。
- 生成内容保存到 `contents`，meta 标记平台和热点来源。

## T4 编辑器能力升级

目标：编辑器具备小说平台排版、章节选择、片段改写、整章重写、自动七维评分和下一章规划。

状态（2026-07-14）：已完成整章重写按钮、章节选择、AI 操作后七维评分与下一章规划接线；待浏览器 E2E 验证长文本排版和错误提示。

开发项：

- 保留章节下拉选择，优化正文阅读/编辑排版。
- 选区浮层支持润色、改写、去 AI 味、续写。
- 工具栏新增“整章重写”按钮。
- 润色、改写、去 AI 味、整章重写后自动调用七维评分和下一章规划。

验收：

- 无选区时片段操作不触发；整章重写对全文生效。
- AI 失败时前端显示错误，不替换正文。
- 成功后展示评分、问题和下一章建议。

## T5 仿写模块

目标：支持基于原文链接或上传/粘贴文本做文风学习和原创仿写。

状态（2026-07-14）：已完成 `/api/v1/imitation` 与 Studio 入口；已加 HTTPS/内网/文本长度防护；后续仍需补相似度红线和版权风险提示。

开发项：

- `POST /api/v1/imitation` 支持 `source_text` 或 HTTPS `source_url`。
- 链接抓取拒绝内网、非 HTTPS、非文本内容。
- 输出包含 `style_profile` 与原创样稿，并保存为 `imitation_sample`。
- Studio 增加“仿写”入口。

验收：

- 少于 200 字输入返回 422。
- 内网 URL 返回 422。
- AI 失败返回 502，不保存空样稿。

## T6 自审与回归

目标：每轮开发完成后 AI 自行审查，确保无明显 bug 和能力断点。

状态（2026-07-14）：新增硬化回归覆盖仿写红线、热点生成、书库详情、出海翻译项目归属、叙事抽取错误暴露、平台连接可视化配置，以及最新审计报告指出的伪热点、伪书本分析、预算空占位、发布凭据断链、融合状态误报、Provider 诊断伪成功、移动端响应式、env 示例缺项。真实 Provider 新功能冒烟 `3 passed, 33.00s`；全量后端 `490 passed, 8 skipped, 1 warning, 110.47s`；前端 `npm run build` 通过；浏览器 E2E `4 passed, 23.0s`（含书库详情、平台连接可视化填写、真实 AI 分析→建书入库）。密钥扫描：Git 跟踪文件无 `sk-` 开头开发 key，`.env.local` 与 `.local/` 均为 ignored。

检查清单：

- 后端：`pytest` 关键用例通过，至少覆盖新增接口。
- 前端：`npm run build` 通过。
- 搜索：确认产品代码无 mock 内容生成路径。
- 数据库：新增查询不破坏新库迁移和已有数据。
- 体验：主要按钮有失败提示，长文本不溢出，详情页可返回。

完成定义：

- 所有任务状态为完成。
- 自审报告记录命令、结果、残余风险。
- 真实 provider smoke 若因本地无 key 未执行，必须明确说明“未执行”，不得算作通过。

## T7 创意拆解、忠实度审计与人工选名

目标：把“粗略创意 → 专业长篇拆解 → 多候选书名 → 用户决定”的顾问流程固化为所有书籍共用的正式主链，禁止策划阶段擅自改动原始需求。

状态（2026-07-16）：已接线；待生产真实 Provider 重跑和浏览器视觉验收后再提升状态。

开发项：

- `plan_idea` 统一输出不可变事实、可选补强、禁止改动、创作圣经、长篇阶段、目标篇幅、内容配比和结构化 `downstream_deliverables`；分卷总纲、前 N 章细纲、章节正文等只登记为后续模块交付物，不在策划节点伪造已产出。
- 新增独立真实 AI 任务 `audit_plan_fidelity`，逐项对比用户原始需求与规划结果；矛盾/遗漏自动反馈给策划 AI 重新生成，最多三轮，仍不合格明确失败。
- 候选书名必须为 3–10 个；排行榜或 AI 建议标题只能作为候选参考，不能自动确认。
- 人工选名页支持选择候选、重新生成候选、填写重新生成反馈、自定义书名；确认前不得进入蓝图或章节写作。
- 策划、忠实度审计、重新生成书名都必须调用 Gateway 真实 Provider并写入 `ai_calls`，失败直接报错。

验收：

- 故意让规划改动用户明确年龄、设备、目标字数或内容主次时，忠实度审计必须拒绝，不能进入 `human_confirm_title`。
- 合格规划必须以 `passed=true`、`score=100`、零 contradictions、零 omissions 才能进入人工选名。
- 用户未确认书名前，工作流状态必须为 `waiting_human`；候选、重生成和自定义书名三个入口都可用。
- 回归证据：相关测试 `38 passed`；全量后端 `557 passed, 9 skipped`。真实生产 Provider 与浏览器证据部署后补录。
