## 一、文档清单与阅读顺序

### 开发前强制阅读（所有 AI/人工开发者）

每次开发、整改、审计修复、提交前，必须先阅读并遵守：

1. `23-AI开发边界与交付真实性规范.md` —— 诚信门禁、状态口径、反撒谎扫描、交付格式。
2. `PROJECT_PROGRESS.md` —— 当前唯一权威进度摘要。
3. 当前执行队列文档（例如 `37-新增需求任务分解-20260713.md`）。
4. 最新审计报告/复审计报告。
5. 仓库根目录 `AGENTS.md` —— AI Agent 最短操作合约。
6. `scripts/ai_development_gate.sh` —— 可执行门禁脚本。

未阅读或未按《23》执行时，任何“完成/融合完成/已验收”声明无效，必须降级为“已接线”或“未真实验收”。

## 三、核心架构速览

```
React + Vite + TS（PWA · 设计系统 · Tiptap 三栏工作台）
        │  REST + SSE
FastAPI（api → service → repository · JWT + Owner/Editor/Viewer）
        │
├── PostgreSQL + pgvector   唯一事实源
├── Redis                   Celery broker · 限流 · 预算 · 缓存 · SSE
├── Celery + beat           五队列 + 定时自动化
├── AI Gateway（进程内）      4 家 Provider 路由/降级/预算/校验/追踪
└── Nginx + HTTPS · Docker Compose · 单 VPS
```

8 大平台能力：C1 内容 · C2 工作流 · C3 AI Gateway · C4 知识 · C5 版本 · C6 叙事一致性 · C7 发布 · C8 追踪。
交付节奏：M1 扫榜成书+热点生产+地基 → M2 长篇小说引擎 → M3 内容工作室 → M4 发布数据出海 → M5 协作离线多端（约 9~10 个月）。
