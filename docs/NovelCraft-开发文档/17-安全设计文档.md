## 1. 威胁模型（STRIDE 简版）与信任边界

### 1.1 资产清点

| 资产 | 敏感度 | 说明 |
|---|---|---|
| 用户内容（小说/章节/知识） | 高 | 唯一事实源在 pg |
| AI API Key / 平台账号 | 极高 | 直接产生成本与被封号风险 |
| 预算与成本数据 | 中 | 影响计费与 DoS |
| 审计日志 | 高 | 取证依据，不可篡改 |
| 用户密码 / JWT | 极高 | 身份根 |
| 备份数据 | 高 | 含全量内容，需加密异地 |

### 1.2 STRIDE 简版

| 类别 | 针对点 | 缓解（对应章节） |
|---|---|---|
| **S**poofing 仿冒 | 伪造 JWT / 盗用 API Key | §2 JWT 短期 + refresh 轮换；§3 RBAC；§4 Key 加密 |
| **T**ampering 篡改 | 篡改内容/版本/审计 | §3 归属过滤；§8 审计不可改；§5 输入校验 |
| **R**epudiation 抵赖 | 越权操作无痕 | §8 audit_logs 中间件级全记录 |
| **I**nfo Disclosure 泄露 | SQL 注入 / 越权读 / XSS | §5 注入防护、转义；§3 水平越权过滤 |
| **D**oS 拒绝服务 | 成本型 DoS / 队列压垮 | §6 预算熔断；§9 限流/healthz |
| **E**levation 提权 | Editor→Owner / Agent 越权调工具 | §3 三角色矩阵；§6 工具白名单 |

### 1.3 信任边界

```
[浏览器 PWA] --HTTPS/JSON+JWT--> [FastAPI 边界]
   边界: 认证校验 / CSRF / 限流 / 审计中间件
        |
   [service 层]  ← 业务逻辑，禁止直连 DB 裸 SQL
        |
   [repository 层] ← ProjectScopedRepository 强制归属过滤
        |                (防水平越权)
   [PostgreSQL + pgvector]  [Redis]  [Celery]
        |
   [AI Gateway] --HTTPS--> [外部 LLM Provider]  (唯一可信外联，白名单)
   [发布适配器] --HTTPS--> [平台 API / Playwright] (显式开启，强约束)
```

- **边界内（可信）**：service/repository、内部 pg/redis、Celery worker。
- **边界外（不可信）**：浏览器输入、外部 LLM、第三方平台、网页抓取源、用户上传文件。
- **外联白名单**：仅 LLM Provider、已配置平台 API、SSRF 白名单域名可出网。

## 2. 认证与会话

### 2.1 JWT 双令牌（短期 access + refresh 轮换）

```python
# app/auth/jwt.py
import jwt, datetime
ACCESS_TTL = 15 * 60            # 15 分钟
REFRESH_TTL = 7 * 24 * 3600     # 7 天，可撤销

def issue_tokens(user_id: int) -> tuple[str, str]:
    now = datetime.datetime.utcnow()
    access = jwt.encode(
        {"sub": str(user_id), "typ": "access",
         "iat": now, "exp": now + ACCESS_TTL, "jti": uuid4().hex},
        settings.JWT_SECRET, algorithm="HS256")
    refresh = jwt.encode(
        {"sub": str(user_id), "typ": "refresh",
         "iat": now, "exp": now + REFRESH_TTL, "jti": uuid4().hex},
        settings.JWT_REFRESH_SECRET, algorithm="HS256")
    # refresh jti 写入 Redis（可吊销）
    redis.setex(f"refresh:{user_id}:{jti}", REFRESH_TTL, "1")
    return access, refresh

def rotate_refresh(user_id, old_jti):
    # 轮换：旧 jti 进黑名单，签发新对
    redis.delete(f"refresh:{user_id}:{old_jti}")
    redis.setex(f"black:{user_id}:{old_jti}", REFRESH_TTL, "1")
    return issue_tokens(user_id)
```

- access 仅 15 分钟；前端无感用 refresh 轮换；
- refresh jti 存 Redis，登出/异常可即时吊销；
- 黑名单（black）用于已注销但未过期的 refresh。

### 2.2 密码哈希

```python
# app/auth/password.py
from passlib.hash import argon2
# 或 bcrypt：passlib.hash.bcrypt
def hash_password(pwd: str) -> str:
    return argon2.using(rounds=4, memory_cost=65536, parallelism=2).hash(pwd)

def verify_password(pwd: str, hashed: str) -> bool:
    return argon2.verify(pwd, hashed)
```

- 默认 argon2（内存硬）；可调 bcrypt；
- 强制密码复杂度策略（长度 ≥10，含大小写+数字）；
- 永不明文存储/日志打印。

### 2.3 登录限流

```python
# 基于 Redis 滑动窗口，防爆破
def login_rate_limit(ip: str, user: str) -> bool:
    key = f"login_rl:{ip}:{user}"
    cnt = redis.incr(key)
    if cnt == 1:
        redis.expire(key, 900)        # 15 分钟窗口
    if cnt > 10:
        raise HTTPException(429, "登录尝试过多，请稍后再试")
    return True
```

- 连续失败 5 次 → 锁定 15 分钟；
- 全局 + 单账号双维度限流；
- 触发告警（Telegram）。

### 2.4 登出与令牌吊销

- 登出：删 Redis `refresh:{uid}:{jti}`，前端清 token；
- 改密/异常：批量吊销该用户所有 refresh（删前缀 + 进黑名单）；
- 中间件对 access 失效、refresh 黑名单做快速拒绝。

## 3. 授权（RBAC 三角色）

### 3.1 三角色矩阵

| 能力 | Owner | Editor | Viewer |
|---|---|---|---|
| 读内容/知识/版本 | ✔ | ✔ | ✔ |
| 编辑内容 / 跑工作流 | ✔ | ✔ | ✘ |
| 创建/恢复版本 | ✔ | ✔ | ✘ |
| 删项目 | ✔ | ✘ | ✘ |
| 管理成员/邀请 | ✔ | ✘ | ✘ |
| 改预算 / model_routes | ✔ | ✘ | ✘ |
| 开启全自动发布 | ✔ | ✘ | ✘ |
| 敏感词表 / 发布 adapter 变更 | ✔ | ✘ | ✘ |

### 3.2 API 层装饰器校验

```python
# app/auth/decorators.py
def require_role(*roles: Role):
    def wrapper(fn):
        @wraps(fn)
        async def inner(request, *a, **kw):
            user = request.state.user
            if user.role not in roles:
                audit_denied(user, request.path, fn.__name__)
                raise HTTPException(403, "权限不足")
            return await fn(request, *a, **kw)
        return inner
    return wrapper

@router.delete("/api/projects/{pid}")
@require_role(Role.OWNER)
async def delete_project(pid, request): ...
```

- 所有写/管理接口显式标注 `@require_role`；
- 缺失装饰器的受保护路由在 CI 静态扫描中报错（见 §9 / 07 §4）。

### 3.3 ProjectScopedRepository 强制归属过滤（防水平越权）

```python
# app/repository/base.py
class ProjectScopedRepository:
    def __init__(self, db, user):
        self.db = db
        self.user = user

    def _scope(self, query):
        # 强制：只返回当前用户 project_members 可见的 project 数据
        member_pids = select(ProjectMember.project_id).where(
            ProjectMember.user_id == self.user.id)
        return query.where(Content.project_id.in_(member_pids))

    def get_content(self, cid):
        return self._scope(select(Content)).where(Content.id == cid).first()
```

- 所有跨用户实体读取经此仓库；**禁止**在 service 直写裸 `select(Content)`；
- 越权访问返回 404（不泄露"存在但无权限"）；
- CI 守卫：扫描 service 层是否绕过 ProjectScopedRepository 直接访问跨项目数据。

## 4. 密钥与凭据管理

### 4.1 API Key / 平台账号 Fernet 加密存储

```python
# app/security/vault.py
from cryptography.fernet import Fernet
import base64, os

def get_fernet() -> Fernet:
    key = os.environ["FERNET_KEY"]          # 仅环境变量注入，不入代码
    return Fernet(base64.urlsafe_b64decode(key))

def encrypt_secret(plain: str) -> str:
    return get_fernet().encrypt(plain.encode()).decode()

def decrypt_secret(token: str) -> str:
    return get_fernet().decrypt(token.encode()).decode()
```

- 所有 LLM Provider Key、平台账号密码/令牌经 Fernet 加密后入库；
- 明文仅存于内存解密瞬间，用完即弃；
- 读取走 `decrypt_secret`，禁止缓存明文到日志/响应。

### 4.2 密钥轮换

- `FERNET_KEY` 支持双 key 过渡（新旧并行解密，逐步迁移）；
- LLM Provider Key 泄露可单点轮换，不影响其它；
- 轮换操作写 audit_logs + 需 Owner 二次确认（§8）。

### 4.3 环境变量与 secrets 管理

- 生产密钥仅由部署环境变量 / Docker secret 注入，**禁明文入库**；
- `.env` 仅含非敏感配置，模板进仓库（`.env.example`）；
- CI 用独立的测试密钥，与生产隔离；
- 密钥扫描：CI 接入 `gitleaks` / `trufflehog`，提交含密钥即阻断。

### 4.4 禁明文入库 / 日志

- 密码：argon2 哈希，永不存明文；
- 凭据：仅密文；
- 日志脱敏：JWT、Key、token、密码字段在 logging formatter 中自动打码（`***`）；
- `ai_calls` 等追踪表只存输入/输出文本，不存密钥。

## 5. 输入输出安全

### 5.1 SQL 注入防护

- 全栈用 SQLAlchemy ORM **参数化查询**，禁止字符串拼接 SQL；
- 原生 SQL（如 pgvector 检索）必须参数绑定，禁止 f-string 拼接；
- **CI 裸 SQL grep 守卫**：

```bash
# .github/ci/sql_guard.sh
# 禁止出现字符串拼接的 SQL（% / .format / f"")
if grep -rnE "(f\"|f'|%s|format\().*(SELECT|INSERT|UPDATE|DELETE)" app/ ; then
  echo "ERROR: 检测到可能的裸 SQL 拼接"
  exit 1
fi
```

- 定期对 ORM 生成语句做注入用例测试（安全套件 INJ-*）。

### 5.2 XSS 防护（Tiptap schema 白名单 + 输出转义）

```typescript
// 前端 Tiptap 白名单 schema
const editor = new Editor({
  extensions: [
    StarterKit.configure({ /* 限定可用节点 */ }),
    // 禁止 raw HTML 节点、script、iframe、on* 属性
  ],
});
// 渲染/输出：React 默认转义；Tiptap JSON 经 sanitize 后再转 HTML
import DOMPurify from "dompurify";
function renderContent(json) {
  const html = editor.schema.toDOM(json);
  return DOMPurify.sanitize(html, { FORBID_TAGS: ["script","iframe"], FORBID_ATTR: ["onerror","onload"] });
}
```

- 编辑态：Tiptap 仅接受白名单节点/marks，禁止任意 HTML；
- 展示态：React 自动转义 + DOMPurify 二次清洗；
- AI 产出内容同样经清洗后再渲染。

### 5.3 CSRF 防护

```python
# 同源 + SameSite=Lax/Strict；敏感写操作要求 CSRF token
# cookie 设置
response.set_cookie("csrf_token", token, httponly=False,
                    samesite="strict", secure=True)
# 前端 fetch 带 header X-CSRF-Token
@app.middleware("http")
async def csrf_guard(request, call_next):
    if request.method in ("POST","PUT","DELETE","PATCH"):
        if request.headers.get("X-CSRF-Token") != get_csrf(request):
            raise HTTPException(403, "CSRF 校验失败")
    return await call_next(request)
```

- 双重：SameSite Cookie + 自定义 CSRF Header（JWT 无 cookie 场景以 Authorization 头为主，CSRF 主要针对凭据型请求）；
- PWA 同源部署，无第三方域写入。

### 5.4 文件上传校验

```python
ALLOWED_TYPES = {"image/png","image/jpeg","image/webp","application/pdf",
                  "text/markdown","application/msword",
                  "application/vnd.openxmlformats-officedocument.wordprocessingml.document"}
MAX_SIZE = 20 * 1024 * 1024   # 20MB

def validate_upload(file):
    if file.content_type not in ALLOWED_TYPES:
        raise HTTPException(400, "不支持的文件类型")
    if file.size > MAX_SIZE:
        raise HTTPException(400, "文件过大")
    # 二次校验 magic bytes，防伪造 content_type
    if not sniff_magic(file):
        raise HTTPException(400, "文件头校验失败")
```

- 类型白名单 + 大小限制 + magic bytes 双重校验；
- 上传文件存对象存储/本地隔离目录，不执行、不返 HTML。

### 5.5 SSRF 防护（网页抓取白名单）

```python
# Knowledge Hub 网页抓取 / 热点采集
SSRF_ALLOWLIST = ["news.example.com", "tech.example.org", ...]
def safe_fetch(url: str):
    host = urlparse(url).hostname
    ip = socket.gethostbyname(host)
    if not in_allowlist(host) or is_private_ip(ip):
        raise HTTPException(400, "目标域名/地址不在白名单")
    # 仅 HTTPS，超时短，禁重定向到内网
    return requests.get(url, timeout=10, allow_redirects=False)
```

- 域名白名单 + 私网 IP 拦截（127.0.0.0/8、10/172/192.168、169.254 元数据）；
- 禁跟随重定向到内网；短超时；
- 抓取结果只作知识入库，**不进分发/发布产物**（合规 §7）。

## 6. AI 特有安全

### 6.1 Prompt 注入防护

- 用户输入与系统 Prompt **严格分离**：用户变量经占位符渲染，不拼接进 system 段；
- 输出做"指令残留"检测（如出现在 `<system>`、忽略上文 等模式）；
- 工作流固定 DAG，Agent 不可被用户输入动态改写节点拓扑（§6.2 工具白名单）；
- 高风险节点（发布/预算）前强制 human 确认。

### 6.2 越权工具调用防护（Agent 工具白名单）

```python
# app/agents/registry.py
@dataclass
class AgentSpec:
    role_prompt: str
    task_prompt_name: str
    tool_whitelist: list[str]   # 默认不含 "call_other_agent"
    input_schema: type
    output_schema: type
    max_retries: int
    budget_scope: str

# 引擎只能经 registry 实例化；agents/** 禁止互相 import（CI 守卫）
def instantiate(agent_name: str) -> AgentNode:
    spec = AGENT_REGISTRY[agent_name]
    return AgentNode(spec)
```

- 工具白名单默认不含"调用其它 Agent"（禁自治对话循环）；
- 想触发另一 Agent 必须由引擎按 DAG 边推进；
- `agents/**` 互相 import 被 CI 静态守卫拦截（02 §13.1）；
- Producer 只产出 WorkflowPlan 数据，不参与运行期（02 §4.2）。

### 6.3 内容安全前置过滤

```python
# 发布网关统一拦截点
def content_safety_gate(content: str, platform: str) -> GateResult:
    if hits_sensitive_words(content):           # 敏感词表
        return GateResult(blocked=True, reason="sensitive_word")
    if not platform_rule_check(content, platform):  # 平台规则库
        return GateResult(blocked=True, reason="platform_rule")
    if not style_similarity_passed(content):     # 仿写产物闸（§6.4）
        return GateResult(blocked=True, reason="style_unchecked")
    return GateResult(passed=True)
```

- 发布前强制过：敏感词表 + 平台规则库 + 仿写相似度标记；
- 任一不通过 → 发布网关拒绝，记录 reason；
- 敏感词表变更需 Owner 人工确认（§8）。

### 6.4 仿写侵权三道闸（02 §13.4 / §5.1）

| 闸 | 位置 | 规则 |
|---|---|---|
| 入库闸 | 样本入库 | 记录 source_type（原创/授权/公共领域/第三方）；第三方样本只抽统计特征，禁整段原文进 style_card 可注入字段 |
| 生成闸 | 仿写生成 | `sim = max(向量余弦, 归一化5-gram)`；`sim ≥0.75` 强制重写（≤2 轮）；`0.6≤sim<0.75` 标记高风险 + human 确认；`sim<0.6` 放行 |
| 产物闸 | 发布网关 | 仿写产物必须携带"相似度检测通过"标记，否则发布网关拒绝（与 §6.3 同点） |

### 6.5 成本型 DoS（预算熔断）

- 任务/项目/日三级预算，Redis 计数（02 §4.1）；
- 超限任务状态 = `PENDING_BUDGET`，非失败（不雪崩）；
- 单用户并发任务上限 + 限流令牌桶（GW-005）；
- 异常高消耗自动降级（429→备胎→PENDING，GW-002）。

## 7. 数据安全与合规

### 7.1 热点抓取合规

- 网页抓取/热点采集结果仅作 Knowledge Hub 知识入库；
- **明确不进入任何分发/发布产物**（架构 §11 / 安全基线）；
- 采集遵守 robots / 频率限制，避免版权与合规风险。

### 7.2 自动发布 ToS 风险与国内平台强约束

> 定案（02 §13.2 / §8.1）：国内平台自动化违反 ToS，有封号风险。

- 三模式：手动 / 半自动（默认，Playwright 填充 + **人点最终发布**） / 全自动（仅官方 API：Medium/Substack/X/WordPress）；
- 国内平台全自动**默认关闭**；开启需设置页勾选"已知悉条款与封号风险" + 二次确认弹窗 + 写 audit_logs；
- 全自动 workflow 在发布 tool 节点前**强制插入 human 确认节点**（每平台首篇必人工确认）；
- 连续失败（验证码/风控）→ 自动降级半自动 + 告警；
- 每次全自动发布落 publish_records 附风险提示副本。

### 7.3 用户内容隐私

- 内容仅项目成员可见（§3 归属过滤）；
- Viewer 只读，不导出/不转发；
- 用户删除项目 → 级联软删内容 + 触发数据删除流程（§7.5）；
- 不在任何日志/埋点泄露正文。

### 7.4 备份加密异地（RPO≤24h / RTO≤1h）

```bash
# make backup
pg_dump $DATABASE_URL | gzip | \
  openssl enc -aes-256-cbc -salt -out backup_$(date +%F).sql.gz.enc
# 上传异地（对象存储 / 异地 VPS）
rclone copy backup_*.enc remote:nc-backups/
# 恢复演练（发版前）
openssl dec ... | gunzip | psql $RESTORE_DB
```

- 每日加密备份 → 异地，RPO≤24h，RTO≤1h；
- **每月恢复演练**（02 §11）；发版前必做 1 次（MVP §7-7）；
- 备份密钥与生产 `FERNET_KEY` 分离管理。

### 7.5 数据删除

- 用户注销 / 项目删除：软删（标记 deleted_at）→ 宽限期后可硬删；
- 硬删级联清理 contents/versions/ai_calls/derivations/knowledge；
- 备份中残留数据按备份轮转自然过期；
- 删除操作写 audit_logs。

## 8. 审计

### 8.1 audit_logs 模型

```sql
audit_logs(
  id, actor_id,        -- 谁
  acted_at,            -- 何时
  action,              -- 做了什么（create/delete/update/publish/...）
  target_type, target_id,  -- 对什么
  detail JSONB, ip, user_agent
)
```

### 8.2 中间件级记录

```python
@app.middleware("http")
async def audit_middleware(request, call_next):
    response = await call_next(request)
    if request.method != "GET" and hasattr(request.state, "user"):
        AuditLog.create(
            actor_id=request.state.user.id,
            action=request.method + " " + request.url.path,
            target_type=infer_target(request),
            target_id=infer_id(request),
            detail={...}, ip=request.client.host)
    return response
```

- 所有写操作（含越权尝试的 403）全记录；
- 审计表只读追加，禁修改/删除（防 Tampering）；
- 越权拦截（AUTH-001~006）必留痕。

### 8.3 敏感操作人工确认

以下操作需 Owner 二次确认 + 写 audit_logs（07 §4 人工确认清单）：

- 数据库 Alembic 迁移；
- model_routes / 预算配置变更；
- 发布 adapter（尤其国内平台自动化）变更；
- 版本系统写路径重大变更；
- 依赖升级；
- 敏感词表变更；
- 全自动发布开启。

## 9. 运维安全

### 9.1 HTTPS / TLS

- Nginx 终止 TLS（≥1.2，优先 1.3），HSTS 开启；
- 证书自动续期（Let's Encrypt / certbot）；
- 内网组件（pg/redis/celery）不暴露公网，仅 VPS 内网互通。

### 9.2 healthz

```python
@app.get("/healthz")
def healthz():
    # 探活 pg / redis / celery 心跳
    return {"status": "ok"}
```

- Nginx/compose healthcheck 依赖 `/healthz`；
- 失败触发自动回滚上一镜像（07 §7）。

### 9.3 依赖漏洞扫描

- CI 集成 `pip-audit` / `safety`（Python）、`npm audit`（前端）；
- 基础镜像 `docker scan` / trivy；
- 高危漏洞阻断合入，定期更新。

### 9.4 最小权限容器

```yaml
# docker-compose.yml 片段
services:
  api:
    user: "1000:1000"          # 非 root
    read_only: true             # 只读根文件系统
    tmpfs: ["/tmp"]
    cap_drop: ["ALL"]           # 丢弃全部 Linux capabilities
    security_opt: ["no-new-privileges:true"]
    networks: ["internal"]      # 仅内网
```

- 容器非 root、只读根、丢弃 capabilities；
- pg/redis 不映射公网端口；
- 网络分段：前端/API 对外，数据层仅内网。

### 9.5 告警（Telegram）

- 失败任务、预算熔断、登录爆破、发布降级、备份失败 → Telegram Bot 实时推送；
- 告警静默期配置，避免风暴；
- 关键告警需人工 ack。
