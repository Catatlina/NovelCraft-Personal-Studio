# 前后端页面接口巡检（2026-07-18）

> 本记录用于对齐新版前端 UI 与现有后端接口。口径遵守《23-AI开发边界与交付真实性规范》：页面可打开、按钮存在、路由存在不等于业务能力完成；真实 AI/外部平台能力仍以真实 Provider 调用、真实凭据和真实回执为准。

## 1. 扫描范围

- 后端源码路由装载点：182 个（按 `APIRouter` 与 `@app.*` 装饰器源码扫描）。
- 前端直接 API 调用点：以 `frontend/src` React 主应用为准；旧 `frontend/public/nc-api.js` 原型桥接脚本已删除，不再进入构建产物或验收范围。
- 浏览器实测侧边栏页面入口：22 个。
- 浏览器实测登录态：本地开发前端 `127.0.0.1:5273` + 后端 `127.0.0.1:8100`。
- 旧原型入口：`/proto.html` 已停止承载产品页面，改为跳转到真实 React 主入口，避免静态样例被误当作接入完成页面。

## 3. 本轮修复的前后端不一致

1. 新版 UI 中「协作管理」组件存在但侧边栏无入口：已新增入口。
2. `overview/workspace/plugins` 三个 tab 为占位页面：已改为真实数据/真实接口面板。
3. 无当前章节时点击「编辑器」会触发 `Cannot read properties of null (reading 'cached')` 并进入全局错误页：已改为安全空态。
4. 无当前作品时「伏笔看板」「版本树」页面空白：已改为可导航空态。
5. 无 `contentId` 时「多平台分发」仍可选择平台并可能发起空 ID 请求：已禁用分发按钮并显示提示。
6. 书库新版卡片缺少 E2E 稳定语义类；设置页平台连接 label 未绑定输入框；成本页“预算/模型路由”不是 heading：已补齐。
7. Playwright 主链仍按旧 UI 的“按钮式导航/旧占位符/旧设置页签”查找：已按新版侧边栏与设置页结构更新。
8. E2E 连续注册临时用户会打到公共注册限流：新增仅 E2E 后端启动时启用的 `NOVELCRAFT_DISABLE_RATE_LIMIT=1`，生产默认不变。
9. 工作台 `DashboardV2` 原有“星渊纪元/赛博长安/今日生成字数”等静态展示已替换为 `/projects`、`/healthz`、`/stats/overview`、`/ranking/library/books`、`/ranking/snapshots`、`/ranking/topics` 的真实数据和真实空态；保留快捷入口、系统状态、书库、榜单、选题、下一步操作链。
10. 概览保留趋势图、题材分布、最近动态、待办和周报导出，但数据由项目、章节、AI 调用、预算、榜单快照、选题和 analytics 接口推导；不再用旧 `Overview.tsx` 的示例数据作为验收依据。
11. 无项目时的扫榜、书库、灵感、工作流、知识库、协作入口不再空白，统一提供“创建项目并继续”的真实操作路径。
12. 前端预算读取纠正为统一 `api()` 解包后的数组；普通账号读取 admin model-routes 403 时改为受控空状态，避免控制台未处理错误。
13. 发布看板不再只要求手填 Content ID；已读取真实项目与书库内容，优先选择最新章节作为发布对象，并新增 `/api/v1/publish-receipts` 人工发布回执端点。回执可记录平台 URL、状态、阅读/点赞/评论/分享/收益，并写入发布记录与 ROI 看板。
14. `/proto.html` 旧静态原型入口已替换为真实主应用跳转页，不再展示“星渊纪元/赛博长安/青梅煮酒”等样例书籍和只弹 toast 的假按钮。
15. `frontend/Dockerfile` 已从“复制 `proto.html` 为生产首页”改为 Node 多阶段构建并发布 React `dist/`，防止容器/线上环境继续展示旧原型。
16. React 新增“工作区”入口和真实接入矩阵，覆盖 22 个主要页面/操作到 `/api/v1` 接口的映射；GET 类接口实际请求，缺项目/作品/章节或操作触发型能力显示阻断，不再用静态话术冒充成功。

## 4. 验证结果

- 浏览器逐页巡检：22/22 侧边栏页面可打开，未发现“建设中”、全局错误页、TypeError、404/500 文案。
- 浏览器实测旧入口：访问 `http://127.0.0.1:5273/proto.html` 自动跳转到 `http://127.0.0.1:5273/`，标题为 `NovelCraft Personal Studio`，页面显示真实主应用登录入口，不再停留在旧静态原型。
- 浏览器实测 React 工作区：注册后进入 `工作区 → 真实接入矩阵`，矩阵可见；15 个接口显示“接口可访问”，6 个显示缺作品/章节或操作触发型阻断；页面正文无“星渊/赛博/青梅/当前原型/ncToast”旧假数据。
- `npm --prefix frontend run build`：通过。
- `set -a; source .env.local; set +a; npm --prefix frontend run test:e2e`：5 passed，1.3m（含真实 Provider 主链②：AI 分析→原创选题→建书入库）。
- `set -a; source .env.local; set +a; cd backend && NOVELCRAFT_TEST_REDIS_URL=redis://localhost:6379/0 .venv/bin/pytest -q tests/test_publish_center.py`：5 passed（含人工发布回执→发布记录→ROI 数据回流）。
- `cd backend && .venv/bin/pytest -q tests/test_ai_failure_truthfulness_regression.py tests/test_truthfulness_gate.py tests/test_v2_bootstrap_wiring.py`：18 passed。
- `python3 scripts/verify_ai_truthfulness.py`：通过。
- `git diff --check`：通过。
- `GATE_ALLOW_WARNINGS=1 bash scripts/ai_development_gate.sh`：退出 0，但仍有历史 warning，交付报告必须说明不能据此宣称全项目完成。
- `rg -n "星渊|赛博|青梅|showToast\\(|onclick=" frontend/proto.html`：无命中；旧静态入口已不再暴露假页面。
- `rg -n "nc-api|当前原型|COPY proto\\.html" frontend/dist frontend/Dockerfile frontend/public`：无命中；生产构建目录和 Dockerfile 不再包含旧原型桥接资源。
- `docker build -t novelcraft-frontend-real-ui:codex frontend`：通过。最终 Dockerfile 不再依赖 `nginx:1.27-alpine`，改用 Node 运行 `server.mjs` 服务 React `dist/` 并代理 `/api`。
- 容器冒烟：`docker run -d --name novelcraft-frontend-smoke -p 127.0.0.1:5280:80 ...` 后 `curl http://127.0.0.1:5280/` 返回 React bundle 的 `index.html`（含 `/assets/index-BpN8vXs_.js`），不是 `proto.html`，也不含 `nc-api.js`。

## 5. 仍不能宣称完成的边界

- 182 个后端路由并非都已有完整浏览器操作路径；本轮重点覆盖侧边栏主入口与主链 E2E。
- 工作台和概览的主要展示已改为真实数据或真实空态；旧 `Overview.tsx` 静态样例组件已删除，`/proto.html` 旧静态原型已下线，`frontend/public/nc-api.js` 原型桥接脚本已删除，三者均不能作为验收依据。
- AI 前端 protected 用例已在 2026-07-19 使用本地忽略文件 `.env.local` 中的 DeepSeek 测试 key 跑通；密钥不得提交，项目截止后回收。
- 平台发布/海外发布/数据回流仍需要真实平台凭据与平台回执，当前只能标记为配置与草稿链路可用。
