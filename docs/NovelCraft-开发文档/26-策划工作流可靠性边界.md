# 策划工作流可靠性边界

## 真实交付范围

Bootstrap 的当前范围是书名、简介、世界观、人物、大纲、第一章和首章审核，不代表整书已经生成。只有后续连续章节流水线完成后，界面才允许使用“整书完成”。

## AI 真实性门禁

- `gen_synopsis / gen_worldview / gen_characters / gen_outline / gen_chapter1` 必须经过严格 Pydantic 输出校验；空对象、缺字段、数量不足或额外字段不能落库为成功。
- 固定 mock 只允许 `NOVELCRAFT_ENV=test` 且显式设置 `NOVELCRAFT_ALLOW_MOCK=true`。普通 development 和 production 缺少真实 Provider 时进入 `pending_provider`。
- mock 输出带 `_meta.synthetic=true`，不得作为真实 Provider 验收证据。
- 每个节点使用稳定 `client_mutation_id`，重放优先读取已经成功的 `ai_calls`，避免重复计费。

## 幂等与恢复

- 榜单小说使用 `ranking-topic:{topic_id}:novel:v1` 生成键。
- 策划运行使用 `ranking-topic:{topic_id}:book-plan:v1` 幂等键。
- 世界观、人物、第一章和版本使用 run/node 级生成键，并依靠数据库部分唯一索引兜底。
- Broker 派发失败写入 `dispatch_failed`、尝试次数和错误；再次请求复用原 run 并重新派发。
- Worker 用数据库条件更新领取节点；已经被其他 Worker 领取的节点不能重复执行。
- BYOK 的 key、base URL 和 model 在每个 Celery 任务开始与结束时清空，禁止跨任务残留。

## 状态语义

- 建书但未运行：小说 `planning`、选题 `planned`。
- Provider 或预算等待：运行和节点显示 `pending_provider/pending_budget`，不生成伪造产物。
- 策划和首章审核通过：运行 `succeeded`、小说 `draft`、选题 `generated`。
- 审核返工耗尽：运行、小说和选题进入 `needs_review`，不得标记成功。

T4 验收仍需要真实 Provider、真实 Worker/Broker 故障、并发请求和数据库对账证据；单元测试与 Mock 不能替代。
