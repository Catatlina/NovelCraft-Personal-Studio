# NovelCraft 服务等级目标（SLO）

> 依据：`novelcraft-remediation-plan.md` P2-T7（真实负载压测 + SLO）。
> 用途：为监控告警与容量规划提供量化目标；压测基线见 `backend/tests/perf/locustfile.py`。

## 1. 核心指标

| 指标 | 目标（SLO） | 测量方式 |
|---|---|---|
| API P95 延迟（读端点） | ≤ 400 ms | 网关访问日志 / APM |
| API P99 延迟（读端点） | ≤ 1200 ms | 网关访问日志 / APM |
| AI 生成类 P95 延迟 | ≤ 30 s（流式首包 ≤ 3 s） | `complete_stream` 首包计时 |
| 错误率（5xx + 4xx 非预期） | ≤ 0.5% | 访问日志聚合 |
| 可用性（月度） | ≥ 99.5% | 健康检查 + 外部探测 |
| 并发读用户 | ≥ 200（单实例，P95 达标） | Locust 基线 |
| AI 并发（受信号量约束） | 按套餐/租户令牌桶，单租户突发不波及他租户 | `circuit_breaker` + 令牌桶 |

## 2. 容量与并发

- **单 Worker 不被单批独占**：批次生成走异步槽位 + 全局 AI 并发信号量（P2-T5）。
- **租户隔离**：单项目风暴触发熔断/限流时，仅影响该项目队列，不波及其他租户（P2-T9）。
- **突发节流**：定时任务（auto_serial 检查等）在高峰加节流（P2-T5）。

## 3. 压测执行

- 基线脚本：`backend/tests/perf/locustfile.py`（只读端点，避免污染数据/配额）。
- 推荐档位：`-u 50 -r 10 -t 2m` 起步，逐步加压至并发目标，记录 P95 / 错误率。
- 门禁：P95 > 400 ms 或错误率 > 0.5% 即视为不达标，需扩容或优化后再上线。
