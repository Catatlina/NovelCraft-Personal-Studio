# 星禾AI工作台 · UI/UX交互指南

> 版本：V1.0 | 日期：2026-07-20 | 角色：UI/UX设计负责人
>
> 本文档规定信息架构、页面布局、交互模式。不是颜色规范（颜色见 DESIGN_SYSTEM.md）。

---

## 一、核心设计原则

| 原则 | 说明 |
|------|------|
| **一屏一个核心任务** | 不要让用户在一屏内处理多件事 |
| **信息分层** | 总览 → 阶段 → 任务 → 详情，渐进深入 |
| **内容优先** | UI是背景，内容才是主角 |
| **减少视觉噪音** | 少用卡片、少用边框、少用分割线 |

## 二、信息架构原则

### 2.1 渐进展示模式

```
❌ 错误：一页展开所有内容
┌──────────────────────────┐
│ 任务1 详情               │
│ 任务2 详情               │
│ 任务3 详情               │  ← 1000行，信息过载
│ 任务4 详情               │
│ ...                      │
└──────────────────────────┘

✅ 正确：信息分层
┌──────────────────────────┐
│ 📊 总览：3/8 任务完成     │  ← 第一层：概览
│ ████████░░░░ 37%         │
│                          │
│ 阶段1：规划 ✓            │  ← 第二层：阶段
│ 阶段2：蓝图 ⏳            │
│ 阶段3：写作              │
│ 阶段4：终审              │
│                          │
│ [展开阶段2]              │  ← 第三层：按需展开
└──────────────────────────┘
```

### 2.2 页面信息量控制

| 页面类型 | 信息量 | 示例 |
|----------|--------|------|
| 概览页 | 3-5个关键指标 | 工作台首页 |
| 列表页 | 20条以内，分页 | 书库、项目列表 |
| 详情页 | 一个对象完整信息 | 小说详情、章节编辑 |
| 编辑页 | 编辑器+辅助面板 | 沉浸式写作 |

## 四、AI交互规范

### 4.1 AI入口

| 入口 | 场景 | 说明 |
|------|------|------|
| ⌘K 命令面板 | 全局 | "AI：续写本章"、"AI：分析市场" |
| 编辑器选中浮条 | 写作中 | 选中文字即浮现AI操作 |
| 右键菜单 | 任意文本 | AI润色/扩写/解释/续写 |
| AI Copilot面板 | 侧边栏 | 对话式交互，带上下文 |
| 实体卡片 | 人物/设定 | AI补全/检查一致性 |

### 4.2 AI交互底线

- **不阻断编辑**：AI操作不弹出模态框，不影响正在编辑的内容
- **显示成本**：每次AI操作显示预估Token消耗
- **可撤销**：AI修改可一键回退
- **流式输出**：所有AI响应默认SSE流式，不等待完整结果
- **加载态**：明确的loading指示 + 进度预估

## 七、移动端适配原则（Phase 2+）

| 原则 | 说明 |
|------|------|
| **一屏一任务** | 比Web更严格，移动端每屏只做一件事 |
| **底部导航** | 5个以内Tab：首页/AI助手/项目/任务/我的 |
| **触控目标** | ≥44px |
| **简化操作** | 查看为主，复杂编辑回PC |
| **不是缩小PC** | 移动端独立设计，不做响应式降级 |
