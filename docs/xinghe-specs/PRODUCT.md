# 星禾AI工作台 · 产品文档

> 版本：V1.0 | 日期：2026-07-20 | 角色：CPO · 高级产品经理

---

## 一、产品愿景

**星禾AI工作台（Xinghe AI Workspace）** 是一个 **AI生产力操作系统**。

不是单一AI工具，而是融合 ChatGPT + Claude + Cursor + Notion + Dify 的多端AI工作平台。

**一句话**：让一个人借助AI，拥有一个小型内容创作团队的生产能力。

---

## 二、目标用户

| 用户画像 | 核心需求 | 当前阶段 |
|----------|----------|----------|
| **网络小说作者** | 扫榜分析 → 选题 → 自动成书 → 去AI味 → 多平台发布 | ✅ Phase 1 |
| **自媒体创作者** | 热点追踪 → 多平台内容生成 → 分发 | 🔜 Phase 2 |
| **知识工作者** | 知识库管理 → AI辅助写作 → 自动化工作流 | 🔜 Phase 3+ |

```
星禾AI工作台

│
└── 扩展层（Extensions）—— V2+
    ├── Skill Center
    ├── Agent Center
    └── Plugin Marketplace
```

## 五、非目标（明确不做什么）

**V1阶段不做：**

- ❌ 社交功能（评论、关注、社区）
- ❌ 电商功能（卖课、商城）
- ❌ 在线协作编辑器（多人实时编辑）
- ❌ Skill/Plugin 公开市场
- ❌ 移动端Native App（Phase 3+）
- ❌ 企业版私有部署
- ❌ 国际化多语言

**这些是V2/V3的目标，不是V1。**

## 六、产品原则

| 原则 | 说明 |
|------|------|
| **先平台，再应用** | 小说是第一个App，架构必须支持后续App扩展 |
| **先核心，再生态** | 先完善平台底座，再做Skill/Plugin市场 |
| **先PC，再移动** | Web是主战场，移动端Phase 2+ |
| **真实数据，拒绝Mock** | 所有功能走真实后端，不做假数据 |
| **证据驱动** | 功能声明必须有测试/截图/验收证据支撑 |

## 七、竞品对标

| 维度 | 对标产品 | 学什么 | 差异化 |
|------|----------|--------|--------|
| 信息密度 | Linear | 紧凑布局、键盘优先 | AI原生操作 |
| 编辑沉浸 | Cursor/Notion | AI浮条、无模式切换 | 小说专用编辑器 |
| 命令交互 | Raycast | ⌘K全局命令 | AI上下文感知 |
| AI能力 | ChatGPT/Claude | 流式对话、多模型 | 扫榜成书闭环 |
| 工作流 | Dify | 可视化编排 | Agent自主执行 |
| 设计语言 | Apple | 简洁高级 | 暗色原生 |
