# 星禾AI工作台 · 安全规范

> 版本：V1.0 | 日期：2026-07-20 | 角色：安全工程师
>
> ⚠️ 安全违规为Block级别，必须修复后才能合并。

---

## 一、认证安全

| 措施 | 说明 |
|------|------|
| **JWT** | Access Token(15min) + Refresh Token(7d) |
| **BYOK** | 用户AI密钥仅内存存储，不落库，关闭会话清除 |
| **密码** | bcrypt哈希，最少8位 |
| **速率限制** | 登录5次/分钟/IP，API 60次/分钟/用户 |
| **会话管理** | 单设备登录（可配），Token黑名单 |

---

## 二、授权安全

| 措施 | 说明 |
|------|------|
| **统一鉴权** | `authz.py` 为唯一鉴权入口 |
| **角色层级** | viewer(0) < editor(1) < owner(2) |
| **资源隔离** | 用户只能访问自己的项目数据 |
| **API Key隔离** | 管理员API不返回密钥原文 |

---

## 三、数据安全

| 措施 | 说明 |
|------|------|
| **传输加密** | HTTPS强制 |
| **平台凭据** | Fernet加密存储 |
| **敏感字段** | 日志脱敏（密钥、密码、Token） |
| **SQL注入** | 参数化查询（已使用%s占位符） |
| **软删除** | DELETE操作使用软删除 |

---

## 四、AI安全

| 措施 | 说明 |
|------|------|
| **Prompt注入防护** | 控制字符过滤 + 注入短语检测 |
| **用户输入清洗** | `sanitize_untrusted()` 统一入口 |
| **输出过滤** | AI输出内容安全检查（敏感词等） |
| **Token预算** | 单次上限5400 token，防止滥用 |
| **断路器** | Provider异常自动熔断 |
| **BYOK不落库** | 用户密钥仅在请求生命周期内存存在 |

---

## 五、代码安全

| 措施 | 说明 |
|------|------|
| **Secret扫描** | CI中gitleaks扫描，禁止提交密钥 |
| **依赖审计** | `npm audit` / `pip-audit` |
| **SAST** | 静态代码安全分析 |
| **代码审查** | 所有PR必须Review |
| **CSP** | Content-Security-Policy头部 |

---

## 六、基础设施安全

| 措施 | 说明 |
|------|------|
| **Nginx** | 反向代理，server_tokens off |
| **Docker** | 非root用户运行，最小基础镜像 |
| **防火墙** | 仅暴露80/443端口 |
| **备份** | 每日自动备份，保留7天 |
| **日志** | 集中收集，敏感信息脱敏 |

---

## 七、安全扫描清单

CI/CD必须通过：

```
□ Secret扫描（gitleaks）
□ 依赖漏洞扫描（npm audit / pip-audit）
□ SAST静态分析
□ CSP头部检查
□ SQL注入检查
□ XSS检查
□ CSRF Token检查
```

---

## 八、安全事件响应

| 级别 | 示例 | 响应时间 | 处理 |
|------|------|----------|------|
| **P0-严重** | 数据泄露、密钥泄露 | 1小时 | 立即修复+通知用户 |
| **P1-高** | 权限绕过、注入漏洞 | 4小时 | 紧急修复 |
| **P2-中** | 配置错误、弱加密 | 24小时 | 计划修复 |
| **P3-低** | 信息泄露、日志过多 | 1周 | 排期修复 |

---

## 九、禁止事项

| ❌ 禁止 | 说明 |
|--------|------|
| 提交API Key/密码/Token到仓库 | gitleaks拦截 |
| 日志中打印密钥 | 日志脱敏 |
| 明文存储凭据 | Fernet加密 |
| 测试环境使用生产数据 | 数据隔离 |
| 硬编码加密密钥 | 环境变量注入 |
| 跳过安全扫描 | CI门禁强制 |
